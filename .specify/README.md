# Specify 目录说明

本目录包含项目的所有治理、规划、规范和任务文档。

## 目录结构

```
.specify/
├── memory/              # 项目记忆和治理
│   └── constitution.md  # 项目宪章（核心治理文档）
├── templates/           # 文档模板
│   ├── plan-template.md
│   ├── spec-template.md
│   ├── tasks-template.md
│   └── commands/        # 命令模板
│       ├── constitution.md
│       ├── plan.md
│       ├── spec.md
│       └── tasks.md
├── plans/               # 项目计划文档
├── specs/               # 详细规范文档
└── tasks/               # 任务分解文档
```

## 文档类型说明

### Constitution（宪章）
- **位置：** `memory/constitution.md`
- **目的：** 定义项目的核心原则和治理规则
- **更新频率：** 低（仅在重大原则变更时）
- **版本控制：** 语义化版本（MAJOR.MINOR.PATCH）

### Plans（计划）
- **位置：** `plans/[feature-name]-plan.md`
- **目的：** 高层次战略规划，定义目标、架构和实施策略
- **适用时机：** 新功能、重大重构、跨团队项目
- **包含内容：** 
  - 宪章合规检查
  - 目标和成功指标
  - 架构设计
  - 测试策略
  - 性能考虑
  - 风险和依赖

### Specs（规范）
- **位置：** `specs/[feature-name]-spec.md`
- **目的：** 详细技术规范，定义所有需求和设计细节
- **适用时机：** 所有非平凡功能实施前
- **包含内容：**
  - 详细功能需求和验收标准
  - 非功能需求（性能、可访问性、安全）
  - 用户体验设计
  - 技术设计（API、数据模型）
  - 全面测试要求

### Tasks（任务）
- **位置：** `tasks/[feature-name]-tasks.md`
- **目的：** 将规范分解为可执行的任务
- **适用时机：** 规范完成后，实施开始前
- **包含内容：**
  - 原子化任务列表
  - 任务分类和优先级
  - 宪章合规检查清单
  - 依赖关系和关键路径
  - 进度跟踪

## 使用工作流

```
1. 制定宪章 (constitution.md)
   ↓
2. 创建计划 (plans/*.md)
   ↓
3. 编写规范 (specs/*.md)
   ↓
4. 分解任务 (tasks/*.md)
   ↓
5. 实施执行
   ↓
6. 审查和交付
```

## 模板使用

所有新文档应基于 `templates/` 目录中的模板创建：

- 使用 `/speckit.plan` 命令创建计划
- 使用 `/speckit.spec` 命令创建规范
- 使用 `/speckit.tasks` 命令创建任务

模板确保：
- 一致的文档结构
- 完整的宪章合规检查
- 必要信息不遗漏
- 文档间正确的交叉引用

## 命名约定

文件命名使用 kebab-case：
- `user-authentication-plan.md`
- `payment-processing-spec.md`
- `dashboard-redesign-tasks.md`

## 版本控制

所有文档应：
- 纳入版本控制（Git）
- 记录创建和修改日期
- 在重大更新时增加版本号
- 保留变更历史（在文档内或通过Git历史）

## 最佳实践

1. **不要跳过步骤** - 即使是小功能，也应至少有简化的规范和任务
2. **保持更新** - 文档应反映实际状态，实施中的发现应更新规范
3. **交叉引用** - 文档间应相互链接，便于追溯
4. **清晰优于简洁** - 宁可过度说明也不要留下歧义
5. **定期审查** - 季度审查宪章和重要文档的相关性

## 工具支持

可以使用以下命令创建和管理文档：

```bash
# 创建或更新宪章
/speckit.constitution [原则描述]

# 创建新计划
/speckit.plan [功能名和需求]

# 创建新规范
/speckit.spec [详细需求]

# 分解任务
/speckit.tasks [基于规范]
```
