# 模板目录

本目录包含所有 Specify 工具包的文档模板。

## 模板列表

### 核心文档模板

1. **plan-template.md** - 项目计划模板
   - 战略级别文档
   - 定义目标、架构、时间线
   - 包含宪章合规检查

2. **spec-template.md** - 技术规范模板
   - 详细设计文档
   - 包含功能和非功能需求
   - 完整的测试要求

3. **tasks-template.md** - 任务分解模板
   - 可执行任务列表
   - 任务分类和优先级
   - 宪章合规检查清单

### 命令模板

`commands/` 目录包含用于生成各类文档的命令模板：

- **constitution.md** - 宪章创建/更新命令
- **plan.md** - 计划创建命令
- **spec.md** - 规范创建命令
- **tasks.md** - 任务分解命令

## 使用指南

### 不要直接编辑模板

模板文件是"主版本"，直接编辑会影响未来所有使用该模板的文档。

### 正确使用方式

通过命令创建基于模板的新文档：

```bash
# 使用模板创建新文档
/speckit.plan [你的需求]
/speckit.spec [你的需求]
/speckit.tasks [你的需求]
```

### 何时修改模板

只在以下情况下修改模板：

1. **宪章更新** - 新原则需要在模板中反映
2. **流程改进** - 发现模板缺少重要部分
3. **标准化** - 团队决定改变文档结构

修改模板时：
- 更新所有相关模板以保持一致性
- 在宪章的"同步影响报告"中记录变更
- 通知团队成员
- 考虑是否需要更新现有文档

## 模板占位符

模板使用方括号包裹的占位符：

```markdown
[PROJECT_NAME]
[FEATURE_NAME]
[YYYY-MM-DD]
[OWNER_NAME]
```

命令工具会自动替换这些占位符。

## 模板设计原则

我们的模板设计遵循以下原则：

1. **完整性** - 涵盖所有必要信息
2. **一致性** - 所有模板结构相似
3. **清晰性** - 明确的指导和示例
4. **宪章对齐** - 强制宪章合规检查
5. **实用性** - 易于使用，不过度复杂

## 模板维护

### 版本控制

模板应与宪章版本保持同步：
- 宪章 MAJOR 版本更新 → 审查所有模板
- 宪章 MINOR 版本更新 → 更新相关模板
- 宪章 PATCH 版本更新 → 通常不需要模板变更

### 质量检查

定期（每季度）审查模板：
- [ ] 是否反映当前宪章原则？
- [ ] 是否包含所有必要部分？
- [ ] 占位符是否清晰？
- [ ] 指导是否足够？
- [ ] 是否有不必要的复杂性？

## 贡献模板改进

如果你发现模板可以改进：

1. 记录问题或建议
2. 与团队讨论
3. 提出具体的改进方案
4. 评估对现有文档的影响
5. 更新相关模板
6. 记录在宪章变更日志中
