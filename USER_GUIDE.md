# 小说RAG分析系统 - 用户使用指南

欢迎使用小说RAG分析系统！本指南将帮助您快速上手并充分利用系统的各项功能。

## 📚 系统介绍

小说RAG分析系统是一个专为中文长篇小说设计的智能分析平台，帮助您：
- 📖 管理和阅读小说
- 🔍 智能搜索小说内容
- 🕸️ 可视化人物关系
- 💡 获得AI驱动的内容理解

## 🚀 快速开始

### 步骤1：启动系统

```bash
cd frontend
npm install
npm run dev
```

在浏览器中访问：http://localhost:5173

### 步骤2：导入第一部小说

1. 点击首页的**"导入新小说"**按钮
2. 准备一个TXT格式的中文小说文件
3. 拖拽文件到上传区域，或点击选择文件
4. 等待系统自动解析和识别章节
5. 填写小说信息并确认导入

## 📖 功能详解

### 1️⃣ 首页 - 小说管理

#### 查看统计信息
首页顶部显示您的阅读统计：
- 已导入小说数量
- 总字数
- 占用存储空间

#### 小说卡片
每部小说显示为一张卡片，包含：
- **书名和作者**
- **字数和章节数**
- **标签**（如果有）
- **导入时间**

#### 操作按钮
每张卡片提供5个操作：
- **分析** - 跳转到搜索页面分析这部小说
- **关系图** - 查看人物关系图谱
- **阅读** - 开始阅读章节内容
- **编辑** - 修改小说信息
- **删除** - 删除小说（需确认）

### 2️⃣ 导入小说

#### 支持的文件格式
- ✅ TXT纯文本文件
- ✅ UTF-8、GBK、GB2312编码

#### 导入流程

**步骤1：上传文件**
- 拖拽TXT文件到虚线框内
- 或点击虚线框选择文件
- 建议文件不超过20MB（约400万字）

**步骤2：文件解析**
系统会自动：
- 检测文件编码
- 统计字数
- 识别章节结构
- 显示预览内容

**步骤3：章节识别**
系统支持识别以下章节格式：
- "第X章"
- "第X回"
- "第X节"
- "第X话"
- "Chapter X"

如果未识别到章节，会按段落自动分段。

**步骤4：填写信息**
- **书名**（必填）- 默认使用文件名
- **作者**（可选）
- **简介**（可选）
- **标签**（可选）- 可添加多个
- **系列设置**（可选）- 如果是系列小说的一部分

**步骤5：确认导入**
- 查看所有信息
- 点击"确认导入"
- 等待处理完成

#### 常见问题

**Q: 文件编码错误怎么办？**
A: 尝试将文件转换为UTF-8编码后重新导入。

**Q: 章节识别不准确？**
A: 目前暂不支持手动调整，建议确保章节标题格式规范。

**Q: 可以导入多少部小说？**
A: 取决于浏览器存储限制，通常可存储数十部小说。

### 3️⃣ 智能搜索与问答

#### 如何搜索

1. **选择搜索范围**
   - 勾选要搜索的小说（可多选）
   - 或选择"全部小说"

2. **输入问题**
   - 支持自然语言问题
   - 例如："张三第一次出现在哪里？"
   - 例如："总结第5章的内容"
   - 例如："XX和YY的关系是什么？"

3. **选择搜索模式**
   - **语义问答**（推荐）- AI理解问题意图
   - **关键词搜索** - 精确匹配关键词

4. **查看结果**
   - 顶部显示AI生成的回答
   - 下方列出相关原文引用

#### 搜索结果说明

**AI回答区域**
- 基于原文生成的回答
- 显示引用的原文数量
- 可复制回答内容
- 可重新生成回答

**参考段落列表**
每个引用包含：
- **来源小说**和**章节信息**
- **段落位置**
- **原文摘录**（关键词高亮）
- **相关度**（百分比或星级）

**操作按钮**
- **展开上下文** - 查看前后文
- **跳转阅读** - 跳到完整章节
- **复制原文** - 复制引用内容

#### 搜索技巧

✅ **DO（推荐）**
- 使用完整的问句
- 提到具体的人物、地点、事件
- 一次问一个问题

❌ **DON'T（不推荐）**
- 使用过于简短的关键词
- 一次问多个不相关的问题
- 使用非中文内容

#### 搜索历史

- 自动保存最近20次搜索
- 点击"历史"按钮查看
- 点击历史记录快速查看之前的结果
- 可清空全部历史

### 4️⃣ 人物关系图谱

#### 查看图谱

1. 在小说卡片点击**"关系图"**
2. 首次生成需要一些时间
3. 之后从缓存快速加载

#### 图谱说明

**节点（圆圈）**
- 代表人物
- 大小 = 出现频率
- 蓝色 = 主要人物
- 绿色 = 次要人物

**连线**
- 代表人物关系
- 颜色区分关系类型：
  - 🔴 红色 = 亲属关系
  - 🔵 蓝色 = 朋友关系
  - ⚫ 灰色 = 敌对关系
  - 🟢 绿色 = 师徒关系
  - 🟡 黄色 = 其他关系
- 粗细 = 关系强度

#### 交互操作

**鼠标操作**
- **拖拽节点** - 调整位置
- **滚轮** - 缩放视图
- **拖拽空白** - 平移视图
- **单击节点** - 高亮关系
- **双击节点** - 查看详情
- **点击连线** - 查看关系详情

**工具栏按钮**
- **放大/缩小** - 控制视图
- **适应屏幕** - 自动调整
- **导出** - 保存为图片

#### 左侧筛选面板

**人物筛选**
- 全部人物
- 仅主要人物
- 仅次要人物

**出现次数**
- 滑块调节最小出现次数
- 过滤不常出现的人物

**搜索人物**
- 输入名称快速定位

**人物列表**
- 按出现频率排序
- 点击人物名高亮图谱中的节点

#### 查看详情

**人物详情**（点击节点）
- 人物名称和类型
- 出现次数统计
- 主要活跃章节
- 关系列表
- 主要出场章节（可点击跳转）

**关系详情**（点击连线）
- 关系双方人物
- 关系类型
- 共同出现章节数
- 代表性原文片段

#### 导出图谱

1. 点击"导出"按钮
2. 选择格式（PNG或JPG）
3. 选择分辨率（1080p/2K/4K）
4. 确认导出

### 5️⃣ 章节阅读

#### 开始阅读

1. 在小说卡片点击**"阅读"**
2. 左侧显示章节目录
3. 右侧显示阅读内容

#### 章节目录

**功能**
- 显示所有章节
- 当前章节高亮
- 显示章节字数
- 搜索章节
- 显示阅读进度

**操作**
- 点击章节切换内容
- 搜索框快速查找章节
- 收起/展开按钮控制显示

#### 阅读区域

**内容显示**
- 章节标题
- 正文内容
- 上一章/下一章导航
- 章节进度条

**跳转功能**
- 从搜索结果跳转会高亮段落
- 高亮效果3秒后自动淡出

#### 阅读设置

点击"设置"按钮打开设置面板：

**字体设置**
- 字体大小：12-22px
- 字体类型：宋体/黑体/楷体/微软雅黑
- 行高：1.4-2.0

**阅读主题**
- **白天模式** - 白底黑字
- **夜间模式** - 深灰底浅字
- **护眼模式** - 淡绿底黑字
- **羊皮纸模式** - 米黄底棕字

**页面宽度**
- 窄（600px）
- 中（800px）
- 宽（1000px）

**其他设置**
- 自动保存阅读进度（开关）

#### 快捷操作

顶部工具栏：
- **A-** - 减小字号
- **A+** - 增大字号
- **设置** - 打开设置面板

## 💾 数据管理

### 数据存储位置

所有数据存储在**浏览器的IndexedDB**中：
- 小说内容
- 搜索历史
- 人物关系图谱
- 阅读设置

### 数据安全

✅ **优点**
- 完全本地化，不上传服务器
- 隐私安全
- 离线可用

⚠️ **注意事项**
- 清除浏览器数据会丢失所有内容
- 不同浏览器不共享数据
- 建议定期导出重要内容

### 存储限制

不同浏览器的存储限制：
- Chrome/Edge: ~可用磁盘60%
- Firefox: ~2GB
- Safari: ~1GB

### 清理数据

**删除单部小说**
1. 在首页找到小说卡片
2. 点击"删除"按钮
3. 确认删除

**清空所有数据**
- 浏览器设置 → 清除浏览数据 → 选择站点数据

## 🎯 使用技巧

### 高效搜索

1. **明确问题**
   - 好：张三第一次出现在第几章？
   - 差：张三

2. **指定范围**
   - 只选择相关的小说进行搜索
   - 提高检索速度和准确度

3. **利用历史**
   - 查看搜索历史重复问题
   - 避免重复搜索

### 深度阅读

1. **结合功能使用**
   - 阅读时遇到问题 → 切换到搜索
   - 搜索到内容 → 跳转到阅读
   - 查看人物关系 → 返回阅读

2. **个性化设置**
   - 长时间阅读使用护眼模式
   - 夜间使用夜间模式
   - 调整舒适的字体大小和行高

3. **利用图谱**
   - 理清复杂的人物关系
   - 查看人物出场章节
   - 跳转到关键章节

## ❓ 常见问题

### Q1: 支持哪些小说格式？
A: 目前仅支持TXT纯文本格式。

### Q2: 可以导入网络小说吗？
A: 可以，下载为TXT格式后导入。

### Q3: 搜索功能是真实的AI吗？
A: 当前版本使用模拟数据演示。实际应用需要接入后端RAG API。

### Q4: 人物关系图谱如何生成？
A: 当前版本使用模拟数据。实际应用需要NLP分析。

### Q5: 数据会丢失吗？
A: 存储在浏览器本地，除非清除浏览器数据或卸载浏览器。

### Q6: 支持手机使用吗？
A: 当前版本主要针对桌面端优化，移动端可访问但体验有限。

### Q7: 可以导出小说吗？
A: 目前不支持导出原文，可复制搜索结果和引用。

### Q8: 如何备份数据？
A: 建议保留原始TXT文件作为备份。
