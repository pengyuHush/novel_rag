# 小说RAG系统使用流程

## 📝 完整使用流程

### 第一步：启动服务

```bash
# 1. 确保 Docker 服务运行
docker-compose up -d

# 2. 启动后端
poetry run uvicorn app.main:app --reload --port 8000

# 3. 访问 API 文档
# http://localhost:8000/docs
```

### 第二步：创建小说记录

**通过 API**:
```bash
curl -X POST http://localhost:8000/api/v1/novels \
  -H "Content-Type: application/json" \
  -d '{
    "title": "我的第一部小说",
    "author": "作者名",
    "description": "这是一部精彩的小说",
    "tags": ["玄幻", "冒险"]
  }'
```

**响应示例**:
```json
{
  "id": "abc123...",
  "title": "我的第一部小说",
  "status": "pending",
  ...
}
```

**记下返回的 `id`，后续步骤需要使用！**

### 第三步：上传小说文件

**准备文件**:
- 格式：TXT（纯文本）
- 编码：UTF-8、GBK 或 GB2312
- 最小字数：1000字
- 中文占比：≥60%
- 最大大小：50MB

**上传**:
```bash
curl -X POST "http://localhost:8000/api/v1/novels/abc123.../upload" \
  -F "file=@/path/to/your/novel.txt"
```

**响应**:
```json
{
  "message": "文件上传成功，正在处理中",
  "novel_id": "abc123...",
  "status": "processing",
  "estimated_time": 120
}
```

### 第四步：等待处理完成

**查询处理状态**:
```bash
curl http://localhost:8000/api/v1/novels/abc123.../status
```

**状态说明**:
- `pending` - 等待上传
- `processing` - 正在处理中
- `completed` - 处理完成 ✅
- `failed` - 处理失败 ❌

**处理过程**:
1. 文件验证（编码、格式、内容）
2. 章节识别
3. 文本分段
4. 向量化（调用智谱 API）
5. 存入 Qdrant

**估计时间**:
- 1万字 ≈ 10秒
- 10万字 ≈ 1-2分钟
- 100万字 ≈ 10-20分钟

### 第五步：RAG 智能搜索

**处理完成后，可以进行搜索**:

```bash
curl -X POST http://localhost:8000/api/v1/search \
  -H "Content-Type: application/json" \
  -d '{
    "query": "主角是谁？",
    "top_k": 5,
    "include_references": true
  }'
```

**响应示例**:
```json
{
  "query": "主角是谁？",
  "answer": "根据小说内容，主角是张三，一个年轻的修仙者...",
  "references": [
    {
      "novel_title": "我的第一部小说",
      "chapter_title": "第一章 开端",
      "chapter_number": 1,
      "content": "张三是一个普通的少年...",
      "relevance_score": 0.95
    }
  ],
  "elapsed": 1.23
}
```

**常见问题类型**:
- 事实性：`主角是谁？`
- 情节性：`第三章发生了什么？`
- 关系性：`张三和李四是什么关系？`
- 总结性：`总结一下前五章的内容`

### 第六步：生成人物关系图谱

```bash
# 触发生成
curl -X POST http://localhost:8000/api/v1/graph/novels/abc123...

# 获取图谱
curl http://localhost:8000/api/v1/graph/novels/abc123...
```

**响应示例**:
```json
{
  "novel_id": "abc123...",
  "characters": [
    {
      "id": "张三",
      "name": "张三",
      "importance": 1.0,
      "occurrences": 245
    }
  ],
  "relationships": [
    {
      "source": "张三",
      "target": "李四",
      "relation": "相关",
      "weight": 0.8,
      "evidence": ["张三和李四在第三章相遇..."]
    }
  ]
}
```

### 第七步：查看章节内容

```bash
# 获取章节列表
curl http://localhost:8000/api/v1/novels/abc123.../chapters

# 获取特定章节内容
curl http://localhost:8000/api/v1/novels/abc123.../chapters/{chapter_id}/content
```

## 🎯 使用技巧

### 1. 搜索优化

**指定小说范围**:
```json
{
  "query": "主角的特点",
  "novel_ids": ["abc123", "def456"],
  "top_k": 3
}
```

**调整返回数量**:
- `top_k`: 1-10（默认5）
- 越大越全面，但可能包含不相关内容

### 2. 文件准备

**清理文本**:
- 删除多余空行
- 统一使用UTF-8编码
- 保持章节标题格式一致

**章节标题格式** (自动识别):
- `第一章 标题`
- `第1章 标题`
- `第001章 标题`
- `第一节 标题`
- `第一回 标题`

### 3. 性能优化

**大文件处理**:
- 文件 > 5MB：分批处理
- 文件 > 10MB：耐心等待
- 文件 > 50MB：不支持

**提升速度**:
- 使用UTF-8编码（避免检测）
- 规范章节标题（减少识别时间）
- 较好的网络连接（智谱API调用）

## ⚠️ 常见问题

### Q1: 搜索返回"知识库为空"

**原因**: 还没有处理完任何小说

**解决**: 
1. 上传小说文件
2. 等待处理完成
3. 再次搜索

### Q2: 文件上传失败

**检查**:
- [ ] 文件是 .txt 格式
- [ ] 文件大小 < 50MB
- [ ] 至少 1000 字
- [ ] 中文字符 ≥ 60%

### Q3: 处理状态一直是 "processing"

**可能原因**:
- 文件很大，需要更多时间
- 智谱 API 响应慢
- 网络问题

**查看日志**:
```bash
# 后端终端会显示处理进度
```

### Q4: 搜索结果不准确

**优化方法**:
- 更具体的问题
- 指定小说ID范围
- 增加 top_k 值
- 使用关键词

## 📚 API 文档

**完整 API 文档**: http://localhost:8000/docs

**核心端点**:
- `POST /api/v1/novels` - 创建小说
- `POST /api/v1/novels/{id}/upload` - 上传文件
- `GET /api/v1/novels/{id}/status` - 查询状态
- `POST /api/v1/search` - RAG 搜索
- `POST /api/v1/graph/novels/{id}` - 生成图谱
- `GET /api/v1/novels/{id}/chapters` - 章节列表


访问: http://localhost:5173
