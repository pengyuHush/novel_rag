# 前后端字段对齐修复文档

## 问题描述

前端使用 TypeScript 驼峰命名（camelCase），后端使用 Python 蛇形命名（snake_case）。需要在后端序列化时将字段名转换为驼峰命名。

## 修复方案

### 1. Schema 层面配置别名（Alias）

为所有 Pydantic 模型添加 `alias` 配置，将蛇形命名映射到驼峰命名：

```python
from pydantic import BaseModel, Field

class NovelSummary(BaseModel):
    id: str
    word_count: int = Field(0, alias="wordCount")
    chapter_count: int = Field(0, alias="chapterCount")
    status: str
    has_graph: bool = Field(False, alias="hasGraph")
    # ... 其他字段
    
    class Config:
        populate_by_name = True  # 允许使用原名和别名
        by_alias = True          # 序列化时使用别名
```

### 2. 自定义 APIRouter

创建自定义的 `APIRouter` 类，默认启用 `response_model_by_alias=True`：

```python
# app/core/router.py
from fastapi import APIRouter as FastAPIRouter

class APIRouter(FastAPIRouter):
    def get(self, path: str, *, response_model_by_alias: bool = True, **kwargs):
        return super().get(path, response_model_by_alias=response_model_by_alias, **kwargs)
    
    # 同样重写 post, put, delete, patch 方法
```

### 3. 更新所有 API 路由

将所有路由文件中的 `APIRouter` 导入更改为自定义版本：

```python
# 修改前
from fastapi import APIRouter

# 修改后
from app.core.router import APIRouter
```

## 注意事项

1. **Pydantic v2**: 使用 `by_alias = True` 在 `Config` 类中
2. **FastAPI**: 需要在路由装饰器中设置 `response_model_by_alias=True`
3. **日期时间**: 确保 datetime 对象正确序列化为 ISO 8601 字符串
4. **嵌套对象**: 子对象也需要配置别名（如 Chapter, Reference 等）
