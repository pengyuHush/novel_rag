# 前后端字段对齐修复完成 ✅

## 问题描述

前端使用 TypeScript 驼峰命名（camelCase），后端使用 Python 蛇形命名（snake_case）。导致前端调用后端 API 时字段无法正确解析。

## 错误表现

1. **前端错误**：
```
TypeError: novels.reduce is not a function
TypeError: Invalid attempt to spread non-iterable instance
```

2. **原因**：
   - 后端返回的数据结构不符合前端预期（字段名不一致）
   - 前端期望直接数组，后端返回分页对象

## 解决方案

### 1. Schema 层配置别名（✅ 已完成）

为所有 Pydantic 模型的蛇形字段添加驼峰别名：

**修改的文件**：
- `app/schemas/novel.py` - NovelSummary, NovelDetail, NovelStatusResponse, NovelProcessingResponse
- `app/schemas/chapter.py` - Chapter, ChapterContentResponse  
- `app/schemas/search.py` - SearchRequest, SearchReference
- `app/schemas/graph.py` - CharacterGraphResponse, CharacterGraphTaskResponse
- `app/schemas/common.py` - Pagination, MessageResponse

**配置方式**：
```python
from pydantic import BaseModel, Field

class NovelSummary(BaseModel):
    word_count: int = Field(0, alias="wordCount")
    chapter_count: int = Field(0, alias="chapterCount")
    has_graph: bool = Field(False, alias="hasGraph")
    imported_at: dt.datetime = Field(..., alias="importedAt")
    updated_at: dt.datetime = Field(..., alias="updatedAt")
    # ... 其他字段
    
    class Config:
        populate_by_name = True  # 允许使用原名和别名
        by_alias = True          # 序列化时默认使用别名
```

### 2. 自定义 APIRouter（✅ 已完成）

创建自定义的 `APIRouter`，默认启用 `response_model_by_alias=True`：

**新建文件**: `app/core/router.py`

**修改的路由文件**：
- `app/api/__init__.py`
- `app/api/v1/__init__.py`
- `app/api/v1/novels.py`
- `app/api/v1/chapters.py`
- `app/api/v1/search.py`
- `app/api/v1/graph.py`
- `app/api/v1/system.py`

### 3. 修复字段映射（✅ 已完成）

**问题**: Schema 定义的是 `imported_at`，但 service 传入的是 `created_at`。

**修改**: `app/services/novel_service.py`
```python
def _to_summary(self, novel) -> NovelSummary:
    return NovelSummary(
        # ...
        imported_at=novel.created_at,  # 将 created_at 映射为 imported_at
        updated_at=novel.updated_at,
    )
```

### 4. 前端 API 适配（✅ 已完成）

**修改**: `frontend/src/utils/api.ts`
```typescript
async getAllNovels(): Promise<Novel[]> {
  if (USE_MOCK_API) {
    return mockNovelAPI.getAllNovels();
  }
  // 后端返回分页结构，提取 data 字段
  const response = await apiRequest<{ data: Novel[]; pagination: any }>('/novels?page=1&page_size=1000');
  return response.data;
}
```

## 关键代码变更总结

1. **所有 Pydantic 模型**: 添加 `Field(..., alias="camelCase")` 和 `Config.by_alias = True`
2. **自定义 APIRouter**: 重写 `get/post/put/delete/patch` 方法，默认 `response_model_by_alias=True`
3. **NovelService**: 将 `created_at` 映射为 `imported_at`
4. **前端 API**: 从分页响应中提取 `data` 字段
