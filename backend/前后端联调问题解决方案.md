# 前后端联调问题解决方案

## 问题一览

### 前端报错 1
```
加载小说列表失败: TypeError: novels.reduce is not a function
    at Object.getStorageInfo (api.ts:401:31)
```

**原因**: 
- `novelAPI.getAllNovels()` 期望返回数组，但后端返回了分页对象 `{ data: [], pagination: {} }`

**解决方案**: 
修改 `frontend/src/utils/api.ts` 中的 `getAllNovels` 方法：

```typescript
async getAllNovels(): Promise<Novel[]> {
  const response = await apiRequest<{ data: Novel[]; pagination: any }>('/novels?page=1&page_size=1000');
  return response.data;  // 提取 data 字段
}
```

### 前端报错 2
```
antd.js:1895 Uncaught TypeError: Invalid attempt to spread non-iterable instance.
In order to be iterable, non-array objects must have a [Symbol.iterator]() method.
```

**原因**: 
- 后端返回的字段名是蛇形命名（snake_case），前端期望驼峰命名（camelCase）
- 前端尝试对非迭代对象使用扩展运算符

**解决方案**: 配置后端统一返回驼峰命名

## 核心解决方案：统一字段命名规范

### 方案概述

使用 Pydantic 的 `alias` 功能 + 自定义 `APIRouter` + FastAPI 的 `response_model_by_alias` 参数。

### 实施步骤

#### 步骤 1: 为所有 Schema 添加别名

为蛇形命名的字段添加驼峰命名的别名：

```python
from pydantic import BaseModel, Field

class NovelSummary(BaseModel):
    id: str
    word_count: int = Field(0, alias="wordCount")
    chapter_count: int = Field(0, alias="chapterCount")
    has_graph: bool = Field(False, alias="hasGraph")
    processing_progress: float = Field(0.0, alias="processingProgress")
    imported_at: dt.datetime = Field(..., alias="importedAt")
    updated_at: dt.datetime = Field(..., alias="updatedAt")
    
    class Config:
        populate_by_name = True  # 允许使用原名或别名进行输入
        by_alias = True          # 输出时使用别名
```

**修改的文件**：
- `app/schemas/novel.py`
- `app/schemas/chapter.py`
- `app/schemas/search.py`
- `app/schemas/graph.py`
- `app/schemas/common.py`

#### 步骤 2: 创建自定义 APIRouter

创建 `app/core/router.py`：

```python
from fastapi import APIRouter as FastAPIRouter
from typing import Any, Callable

class APIRouter(FastAPIRouter):
    """自动启用 response_model_by_alias 的路由器"""
    
    def get(self, path: str, *, response_model_by_alias: bool = True, **kwargs):
        return super().get(path, response_model_by_alias=response_model_by_alias, **kwargs)
    
    def post(self, path: str, *, response_model_by_alias: bool = True, **kwargs):
        return super().post(path, response_model_by_alias=response_model_by_alias, **kwargs)
    
    # 同样处理 put, delete, patch, add_api_route
```

#### 步骤 3: 更新所有路由导入

将所有 API 路由文件中的导入从：
```python
from fastapi import APIRouter
```

改为：
```python
from app.core.router import APIRouter
```

**修改的文件**：
- `app/api/__init__.py`
- `app/api/v1/__init__.py`
- `app/api/v1/novels.py`
- `app/api/v1/chapters.py`
- `app/api/v1/search.py`
- `app/api/v1/graph.py`
- `app/api/v1/system.py`

#### 步骤 4: 修复字段映射

数据库模型使用 `created_at`，但前端需要 `importedAt`。在 service 层做映射：

```python
# app/services/novel_service.py
def _to_summary(self, novel) -> NovelSummary:
    return NovelSummary(
        id=novel.id,
        # ... 其他字段
        imported_at=novel.created_at,  # 将 created_at 映射为 imported_at
        updated_at=novel.updated_at,
    )
```

#### 步骤 5: 前端适配分页响应

```typescript
// frontend/src/utils/api.ts
async getAllNovels(): Promise<Novel[]> {
  const response = await apiRequest<{ data: Novel[]; pagination: any }>('/novels?page=1&page_size=1000');
  return response.data;  // 从分页对象中提取数组
}
```

## 关键技术点

### 1. Pydantic 别名系统

- `Field(..., alias="camelCase")`: 定义别名
- `Config.populate_by_name = True`: 允许使用原名或别名输入
- `Config.by_alias = True`: 序列化时使用别名

### 2. FastAPI 响应模型配置

- `response_model`: 指定响应的 Pydantic 模型
- `response_model_by_alias`: 控制是否使用别名序列化

### 3. 自定义 APIRouter

通过继承 `FastAPIRouter` 并重写 HTTP 方法装饰器，实现全局默认配置。

## 常见问题

### Q: 为什么不直接在数据库模型中使用驼峰命名？
A: Python 社区约定使用蛇形命名，保持数据库字段和 Python 代码风格一致。通过别名实现前后端分离。

### Q: 如果忘记添加别名会怎样？
A: 该字段会使用蛇形命名输出，前端无法识别。建议统一检查所有 Schema。

### Q: `populate_by_name` 和 `by_alias` 的区别？
A: 
- `populate_by_name`: 输入时允许使用原名或别名
- `by_alias`: 输出时使用别名

## 总结

通过以下三个层次的配置，实现了前后端字段命名的自动转换：

1. **Schema 层**: 使用 Pydantic `alias` 定义字段别名
2. **路由层**: 自定义 `APIRouter` 默认启用别名序列化
3. **Service 层**: 处理特殊字段映射（如 `created_at` → `imported_at`）

这种方案：
- ✅ 前后端各自使用自己的命名规范
- ✅ 自动转换，无需手动处理
- ✅ 类型安全，Pydantic 校验
- ✅ 易于维护和扩展
