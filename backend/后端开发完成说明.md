## 🚀 如何启动

### 第一步：确保环境就绪

```bash
# 1. 进入backend目录
cd backend

# 2. 确认Docker服务运行
docker-compose ps
# 如未运行，执行: docker-compose up -d

# 3. 确认已安装依赖
poetry install

# 4. 配置环境变量
cp env.example .env
# 编辑 .env 文件，填入: ZAI_API_KEY=你的智谱AI密钥
```

### 第二步：验证环境

```bash
poetry run python verify_env.py
```

应该看到所有检查项都通过 ✅

### 第三步：启动后端服务

#### 方式1: 使用启动脚本 (Linux/Mac)
```bash
chmod +x start.sh
./start.sh
```

#### 方式2: 手动启动 (Windows/通用)
```bash
poetry run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 第四步：验证服务

启动成功后，访问:
- **Swagger文档**: http://localhost:8000/docs
- **健康检查**: http://localhost:8000/api/v1/system/health

或运行测试脚本:
```bash
poetry run python test_api.py
```

## 🔌 前后端集成

### 前端配置
前端已经配置好API调用 (`frontend/src/utils/api.ts`)，只需要:

1. 确保后端运行在 `http://localhost:8000`
2. 前端会自动连接到后端API
3. CORS已配置允许前端域名

### API基础URL
```typescript
const API_BASE_URL = 'http://localhost:8000/api/v1';
```

### 主要接口对应
- 前端 `createNovel()` → 后端 `POST /api/v1/novels`
- 前端 `uploadNovelFile()` → 后端 `POST /api/v1/novels/{id}/upload`
- 前端 `searchNovels()` → 后端 `POST /api/v1/search`
- 前端 `getCharacterGraph()` → 后端 `GET /api/v1/graph/novels/{id}`

## 🐛 常见问题

### Q1: Poetry install 失败
```bash
# 尝试清除缓存
poetry cache clear pypi --all
poetry install
```

### Q2: Docker服务无法启动
```bash
# 查看日志
docker-compose logs redis
docker-compose logs qdrant

# 重启服务
docker-compose down
docker-compose up -d
```

### Q3: 智谱API调用失败
- 检查API Key是否正确
- 确认账户已充值
- 检查网络连接

### Q4: 文件上传失败
- 确保文件是TXT格式
- 文件大小<50MB
- 中文字符占比>=60%
- 文本长度>=1000字
