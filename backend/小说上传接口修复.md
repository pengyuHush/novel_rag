# 小说上传接口修复

## 问题描述

用户尝试导入小说时报错：

```
ImportNovelModal.tsx:164 导入失败: TypeError: Cannot read properties of undefined (reading 'id')
    at Object.uploadNovel (api.ts:155:38)
```

## 问题分析

### 前端期望的响应格式

前端 `createNovel` 方法（`frontend/src/utils/api.ts:100-104`）期望后端返回：

```typescript
{
  message: string;
  novel: Novel;
}
```

然后通过 `response.novel` 提取小说对象。

### 后端实际返回格式

后端 `POST /api/v1/novels` 接口（`backend/app/api/v1/novels.py:43-49`）直接返回 `NovelDetail`：

```python
@router.post("", response_model=NovelDetail, status_code=status.HTTP_201_CREATED)
async def create_novel(
    data: NovelCreate,
    service: NovelService = Depends(get_novel_service),
) -> NovelDetail:
    """Create a new novel entry."""
    return await service.create_novel(data)
```

**返回示例**：
```json
{
  "id": "abc123",
  "title": "测试小说",
  "author": "作者",
  "wordCount": 0,
  "chapterCount": 0,
  ...
}
```

### 根本原因

前端代码尝试访问 `response.novel.id`，但后端直接返回小说对象（没有 `novel` 包装），导致 `response.novel` 为 `undefined`，进而访问 `.id` 时报错。

## 解决方案

修改前端代码，直接接收 `Novel` 对象，而不是从包装对象中提取：

### 修改文件：`frontend/src/utils/api.ts`

**修改前**：
```typescript
const response = await apiRequest<{ message: string; novel: Novel }>('/novels', {
  method: 'POST',
  body: JSON.stringify(metadata),
});
return response.novel;  // ❌ response.novel 是 undefined
```

**修改后**：
```typescript
// 后端直接返回 NovelDetail，不是 { message, novel } 包装
return apiRequest<Novel>('/novels', {
  method: 'POST',
  body: JSON.stringify(metadata),
});  // ✅ 直接返回 Novel 对象
```

## API 响应格式说明

### 标准响应格式

根据后端实现，不同接口的响应格式如下：

| 接口 | 方法 | 响应类型 | 说明 |
|------|------|---------|------|
| `/novels` | GET | `NovelListResponse` | `{ data: Novel[], pagination: {...} }` |
| `/novels` | POST | `NovelDetail` | 直接返回小说对象 |
| `/novels/{id}` | GET | `NovelDetail` | 直接返回小说对象 |
| `/novels/{id}` | PUT | `NovelDetail` | 直接返回小说对象 |
| `/novels/{id}` | DELETE | `MessageResponse` | `{ message: string }` |
| `/novels/{id}/upload` | POST | `NovelProcessingResponse` | `{ message, novelId, status, estimatedTime }` |

### 规律总结

- **CRUD 操作**：直接返回资源对象（`NovelDetail`、`NovelListResponse` 等）
- **操作消息**：返回包装对象（`MessageResponse`、`NovelProcessingResponse` 等）
