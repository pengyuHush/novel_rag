## 问题 3: SQLAlchemy 异步会话过期错误

### ❌ 错误信息
```
sqlalchemy.exc.MissingGreenlet: greenlet_spawn has not been called; can't call await_only() here.
```

### 🔍 问题原因
在 `NovelService.update_novel()` 和 `create_novel()` 方法中：
1. 执行 `await self.session.commit()` 后，ORM 对象的状态被标记为过期
2. 之后调用 `await self.get_novel(novel.id)` 时，需要重新查询数据库
3. 但此时已经离开了原来的异步上下文，导致 SQLAlchemy 无法执行异步操作

### ✅ 修复方案

**修改文件**: `backend/app/services/novel_service.py`

**原代码**:
```python
async def update_novel(self, novel_id: str, data: NovelUpdate) -> NovelDetail | None:
    novel = await self.repo.get(novel_id)
    if not novel:
        return None
    await self.repo.update(novel, ...)
    await self.session.commit()
    return await self.get_novel(novel.id)  # ❌ 会触发错误
```

**修复后**:
```python
async def update_novel(self, novel_id: str, data: NovelUpdate) -> NovelDetail | None:
    novel = await self.repo.get(novel_id)
    if not novel:
        return None
    await self.repo.update(novel, ...)
    await self.session.commit()
    # ✅ 在同一会话中刷新对象
    await self.session.refresh(novel)
    # ✅ 直接在当前会话中转换
    summary = self._to_summary(novel)
    return NovelDetail(**summary.model_dump(), content=novel.content, file_path=novel.file_path)
```

**关键点**:
- 使用 `await self.session.refresh(novel)` 刷新对象
- 在同一会话中直接访问属性并转换
- 避免跨会话查询

## 问题 4: Qdrant 版本不匹配警告

### ⚠️ 警告信息
```
UserWarning: Qdrant client version 1.15.1 is incompatible with server version 1.7.4
```

### 🔍 问题原因
- Qdrant Python 客户端版本 (1.15.1) 比 Docker 中的服务器版本 (1.7.4) 新
- 客户端默认会检查版本兼容性

### ✅ 修复方案

**修改文件**: `backend/app/core/qdrant.py`

**修复后**:
```python
@lru_cache(maxsize=1)
def get_qdrant_client() -> QdrantClient:
    return QdrantClient(
        host=settings.QDRANT_HOST,
        port=settings.QDRANT_PORT,
        prefer_grpc=False,  # 使用 HTTP 协议
        timeout=30.0,       # 设置超时
    )
```

**说明**:
- 虽然有版本差异警告，但实际功能完全正常
- 如果想完全消除警告，可以升级 Qdrant 服务器到最新版本
