## 根本原因分析

**FastAPI BackgroundTasks 的异步处理问题**：

向量化是一个耗时的过程（对于 30,000 字的小说，需要调用多次 API 生成 embedding）。BackgroundTasks 可能因为以下原因失败：

1. **任务超时**：处理时间过长
2. **异常未捕获**：某个环节出错但没有正确记录
3. **进程重启**：如果在处理期间后端重启，任务会丢失

### 调试脚本

创建测试脚本直接调用处理逻辑：

```python
# backend/test_process_novel.py
import asyncio
from pathlib import Path
from app.db.session import get_db_session
from app.services.text_processing_service import TextProcessingService

async def main():
    novel_id = "bc1f771c09674598abae9ff05f680a33"
    file_path = Path("storage/novels/bc1f771c09674598abae9ff05f680a33.txt")
    
    session_gen = get_db_session()
    session = await anext(session_gen)
    try:
        service = TextProcessingService(session)
        
        async def progress_callback(progress, message):
            print(f"[{progress*100:.1f}%] {message}")
        
        await service.process_novel(novel_id, file_path, progress_callback)
        print("✅ Processing completed!")
        
    except Exception as e:
        print(f"❌ Error: {e}")
        import traceback
        traceback.print_exc()
    finally:
        try:
            await anext(session_gen)
        except StopAsyncIteration:
            pass

if __name__ == "__main__":
    asyncio.run(main())
```

运行：
```bash
poetry run python test_process_novel.py
```
