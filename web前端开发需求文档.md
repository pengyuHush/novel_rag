# 小说RAG系统 - Web前端开发需求

## 项目概述

请帮我开发一个**中文小说智能分析系统**的Web前端界面。这是一个基于RAG（检索增强生成）技术的小说阅读辅助平台，专注于中文长篇小说的智能检索、问答和关系分析。

**重要说明：**

- 当前阶段只需完成前端界面和交互逻辑
- 暂时不涉及后端接口设计，请使用模拟数据（Mock Data）
- 所有数据处理暂时在前端本地完成（使用 IndexedDB 存储）
- 专注于UI/UX设计和用户交互流程

---

## 技术栈建议

**核心框架：**

- React 18+ (推荐使用 TypeScript)
- Vite 作为构建工具

**UI组件库：**

- Ant Design 或 Material-UI（推荐Ant Design，更适合中文界面）
- 或使用 Tailwind CSS + shadcn/ui

**状态管理：**

- Zustand 或 React Context（轻量级项目首选）
- 或 Redux Toolkit（如果需要更复杂的状态管理）

**数据可视化：**

- D3.js 或 Recharts（用于人物关系图谱）
- React Flow 或 Cytoscape.js（网络图展示）

**本地存储：**

- IndexedDB（使用 Dexie.js 封装）
- LocalStorage（用于简单配置和搜索历史）

**文件处理：**

- jschardet 或 encoding-japanese（文件编码检测）
- 原生 File API（文件上传和读取）

**其他工具：**

- React Router（页面路由）
- axios 或 fetch（未来接入后端API预留）
- dayjs（时间处理）

---

## 页面结构和路由

系统包含以下3个核心页面：

```
/                    - 搜索与问答页（包含小说管理侧边栏）
/graph/:novelId      - 人物关系图谱页
/reader/:novelId     - 章节阅读页
```

**说明：** 原首页的小说管理功能整合到搜索页面，作为左侧可收起/展开的侧边栏存在。

## 详细功能需求

### 1. 搜索与问答页（主页） (`/`)

#### 1.1 页面布局

**顶部导航栏：**

- 左侧：系统Logo和名称（"小说RAG分析系统"）
- 右侧：帮助按钮

**左侧小说管理侧边栏（可收起/展开）：**

- 宽度：约 300px
- 可通过按钮收起/展开
- 收起时仅显示图标，展开时显示完整内容

**侧边栏内容：**

- 顶部统计卡片：
  - 显示统计信息：已导入小说数量、总字数、存储空间占用
  - "导入新小说"按钮（醒目）

- 小说列表区：
  - 每部小说显示为一个列表项，包含：
    - 书名（加粗）
    - 作者名（小字）
    - 字数统计（如："约120万字"）
    - 选择复选框（用于选择搜索范围）
  - 悬停时显示操作按钮：
    - "关系图" - 跳转到人物关系图谱页
    - "阅读" - 跳转到章节阅读页
    - "编辑" - 编辑小说信息
    - "删除" - 删除小说（需二次确认）

**主搜索区域（右侧或侧边栏收起时占满）：**

- 大尺寸搜索框
- 搜索选项和设置
- 搜索结果展示区

#### 1.2 导入小说功能

**触发方式：**

- 点击"导入新小说"按钮
- 拖拽TXT文件到指定区域

**导入流程（使用Modal对话框）：**

**步骤1：文件上传**

- 显示拖拽上传区域（虚线边框，带上传图标）
- 提示文字："拖拽TXT文件到此处，或点击选择文件"
- 支持多文件选择（批量导入）
- 文件格式限制：仅.txt
- 文件大小提示："建议不超过20MB"

**步骤2：文件解析和预览**

- 显示上传进度条
- 自动检测文件编码（UTF-8、GBK、GB2312）
- 显示检测结果："已识别编码：UTF-8"
- 显示文件基本信息：
  - 文件名
  - 文件大小
  - 字数统计
- 显示文本预览（前1000字）

**步骤3：章节识别**

- 自动识别章节标题
- 显示识别到的章节列表（可折叠展开）：
  - 章节序号
  - 章节标题
  - 章节字数
- 提供调整选项：
  - 手动添加章节分界
  - 删除错误识别的章节
  - 合并章节
- 如果未识别到章节，提示："未检测到章节标题，将按段落自动分段"

**步骤4：填写小说信息**

- 书名输入框（必填，默认使用文件名）
- 作者输入框（可选）
- 简介文本框（可选，多行）
- 标签输入（可选，支持添加多个标签）
- 系列关系设置：
  - 下拉选择："独立小说" / "系列小说"
  - 如果选择系列，显示：
    - 关联已有小说（多选下拉框）
    - 系列序号（第1部、第2部等）

**步骤5：确认导入**

- 显示所有信息总览
- "确认导入"按钮
- "取消"按钮
- 导入中显示进度条和状态："正在建立索引..."
- 导入完成后显示成功提示，自动关闭对话框，刷新小说列表

#### 1.3 小说选择与管理

**小说列表交互：**

- 点击小说列表项的复选框选择/取消选择该小说
- 选中的小说会自动作为搜索范围
- 支持全选/全不选
- 悬停小说项时显示操作菜单

**编辑小说信息：**

- 点击"编辑"按钮，弹出对话框
- 可修改书名、作者、简介、标签
- 可调整系列关系
- 保存后更新列表显示

**删除小说：**

- 点击"删除"按钮
- 弹出确认对话框："确定要删除《书名》吗？此操作不可撤销。"
- 二次确认按钮："确定删除"（红色）和"取消"
- 删除后显示Toast提示："已删除《书名》"
- 从IndexedDB中删除所有相关数据

### 2. 搜索与问答功能（主页核心功能）

#### 2.1 搜索区域布局

**搜索框：**

- 大尺寸输入框，占据显著位置
- 占位符文本："输入问题或关键词，如：XX角色第一次出现在哪里？"
- 右侧搜索按钮（放大镜图标）
- 支持回车键触发搜索

**搜索选项（搜索框下方）：**

- 搜索范围提示：
  - 显示已选择的小说数量："已选择3部小说进行搜索"
  - 点击可展开/收起左侧小说列表
  - 如果未选择小说，显示提示："请在左侧选择小说"
- 搜索模式切换：
  - Tab切换："关键词搜索" / "语义问答"
- 高级选项（可折叠）：
  - 章节范围筛选（滑块选择）
  - 匹配模式：精确/模糊

**主内容区：**

**默认状态（无搜索结果）：**

- 显示引导文字："开始搜索，探索小说世界"
- 显示示例问题（可点击）：
  - "XX角色是谁？"
  - "总结第5章的内容"
  - "XX和YY的关系是什么？"
- 显示最近的搜索历史（最多5条，可点击重新查看）

**搜索结果状态：**

**1. 回答区域（顶部）**

- 卡片样式，背景色与主背景区分
- 显示AI生成的回答文字
- 字体稍大，易于阅读
- 右上角显示："基于X处原文生成"
- 底部操作按钮：
  - 复制回答
  - 重新生成（使用不同的回答策略）

**2. 参考段落列表（下方）**

- 标题："相关原文引用"
- 每个参考段落显示为一个卡片：
  
  **卡片头部：**
  - 来源小说名称（加粗）
  - 章节信息："第X章 章节标题"
  - 段落位置："第X段" 或 "字符位置：1234-5678"
  - 相关度显示：进度条或星级（如 ★★★★☆ 85%）

  **卡片内容：**
  - 原文摘录（200-500字）
  - 关键词高亮显示（黄色背景）
  - 如果内容被截断，显示"..."

  **卡片底部操作：**
  - "展开上下文"按钮 - 展开显示前后各1-2段
  - "跳转阅读"按钮 - 跳转到章节阅读页的对应位置
  - "复制原文"按钮

**3. 加载更多**

- 底部显示"加载更多"按钮或自动加载

**无结果状态：**

- 显示提示："未找到相关内容"
- 提供建议：
  - "尝试换个说法"
  - "检查搜索范围是否正确"
  - "确认小说已成功导入"

#### 2.2 搜索历史

**搜索历史显示：**

- 在顶部导航栏显示历史按钮
- 点击后弹出抽屉显示历史记录
- 列表显示历史记录（时间倒序）：
  - 问题文本（最多显示一行，超出省略）
  - 搜索时间（如："2小时前"）
  - 点击可重新查看该次搜索结果
- 底部操作：
  - "清空历史"按钮（需确认）

#### 2.3 交互细节

**搜索过程：**

- 点击搜索后显示加载动画
- 显示状态文字："正在分析问题..." → "正在检索相关内容..." → "正在生成回答..."
- 加载时间较长时，显示温馨提示："处理大量文本需要一些时间，请稍候..."

**错误处理：**

- 如果搜索失败，显示错误提示
- 如果未选择小说，提示："请先选择要搜索的小说"
- 如果输入为空，提示："请输入问题或关键词"

**响应式设计：**

- 在较小屏幕下，侧边栏自动隐藏，通过按钮切换显示

### 3. 人物关系图谱页 (`/graph/:novelId`)

#### 3.1 页面布局

**顶部工具栏：**

- 左侧：
  - 返回按钮："← 返回搜索"
  - 当前小说名称显示
  - 如果有多部关联小说，显示下拉选择："查看其他小说的关系图"
- 中间：
  - 缩放控制按钮：放大 / 缩小 / 适应屏幕
  - 重置布局按钮
- 右侧：
  - 导出按钮（导出为PNG/JPG）
  - 全屏按钮

**左侧面板（约20%宽度，可收起）：**

**人物筛选：**

- 标题："人物筛选"
- 筛选选项：
  - 单选："全部人物" / "主要人物" / "次要人物"
  - 滑块："最少出现次数"（调节显示人物的出现频率阈值）
- 人物列表：
  - 显示所有人物名称
  - 按出现频率排序
  - 显示出现次数：如"张三 (158次)"
  - 点击人物名可在图谱中高亮显示
  - 支持搜索人物名

**图例说明：**

- 节点大小：出现频率（大节点=主要人物）
- 关系类型颜色：
  - 红色：亲属关系
  - 蓝色：朋友关系
  - 绿色：师徒关系
  - 灰色：敌对关系
  - 黄色：其他关系

**主内容区（图谱展示区）：**

**关系网络图：**

- 使用力导向图（Force-Directed Graph）展示
- 人物显示为圆形节点：
  - 节点内显示人物名称（中文字体）
  - 节点大小反映出现频率
  - 主要人物节点更大更醒目
- 关系显示为连线：
  - 不同关系类型使用不同颜色和线型
  - 线的粗细反映关系强度
  - 鼠标悬停显示关系类型标签

**交互功能：**

- 拖拽节点：可拖动人物位置
- 缩放：鼠标滚轮或双指缩放
- 平移：拖拽空白区域平移视图
- 点击节点：高亮该人物的所有关系，其他关系半透明
- 双击节点：打开详情面板
- 点击关系线：显示该关系的详细信息

**右侧详情面板（默认隐藏，点击节点/关系线后显示）：**

**人物详情（点击节点时显示）：**

- 人物名称（大标题）
- 基础信息：
  - 出现次数："共出现158次"
  - 主要活跃章节："第1-50章"
- 关系列表：
  - 显示该人物的所有关系
  - 每个关系显示：
    - 关联人物名称
    - 关系类型
    - 点击可跳转到该关系
- 主要出场章节列表（可点击跳转阅读）：
  - "第3章 初次相遇"
  - "第15章 重要转折"
  - ...
- 底部按钮："查看所有相关内容"（跳转到搜索页并自动搜索该角色）

**关系详情（点击关系线时显示）：**

- 关系双方人物名称："张三 ↔ 李四"
- 关系类型："朋友关系"
- 共同出现章节："共在23个章节中同时出现"
- 代表性原文片段（2-3段）：
  - 展示最能体现该关系的原文
  - 显示章节来源
  - 点击可跳转阅读完整章节

#### 3.2 图谱生成和加载

**初次进入（未生成图谱）：**

- 显示提示："正在分析人物关系，请稍候..."
- 显示进度条："已分析 X / Y 章节"
- 分析完成后自动显示图谱

**已有图谱：**

- 快速加载并渲染
- 显示加载动画："加载中..."

#### 3.3 导出功能

点击"导出"按钮：

- 弹出选项对话框：
  - 选择格式：PNG / JPG
  - 选择分辨率：1080p / 2K / 4K
  - 是否包含图例
- 点击"确认导出"生成图片并下载
- 文件名格式："{书名}_人物关系图_{日期}.png"

## UI/UX 设计要求

### 整体风格

**设计理念：**

- 简洁、现代、专业
- 以内容为中心，减少视觉干扰
- 适合长时间使用和阅读

**色彩方案：**

- 主色调：蓝色系（#1890ff）- 代表知识和科技
- 辅助色：灰色系（#f0f2f5背景，#262626文字）
- 强调色：橙色（#fa8c16）- 用于重要按钮和提示
- 成功色：绿色（#52c41a）
- 警告色：红色（#ff4d4f）

**字体：**

- 标题：思源黑体 / 微软雅黑（加粗）
- 正文：思源宋体 / 苹方 / 微软雅黑
- 代码/数据：Monaco / Consolas
- 确保中文字体清晰易读

**间距和布局：**

- 使用8px栅格系统（8、16、24、32等）
- 卡片内边距：24px
- 元素间距：16px
- 页面边距：32px（大屏）/ 16px（小屏）

### 组件设计

**按钮：**

- 主要按钮：蓝色背景，白色文字，圆角4px
- 次要按钮：白色背景，灰色边框
- 危险按钮：红色背景（用于删除等操作）
- 按钮尺寸：小(32px) / 中(40px) / 大(48px)
- 悬停效果：微微加深颜色
- 点击效果：轻微缩放动画

**卡片：**

- 白色背景，淡灰色边框
- 圆角8px
- 轻微阴影：box-shadow: 0 2px 8px rgba(0,0,0,0.08)
- 悬停时阴影加深

**输入框：**

- 边框：1px solid #d9d9d9
- 圆角4px
- 聚焦时边框变蓝色
- 占位符文字浅灰色

**加载状态：**

- 使用Spin组件或骨架屏
- 加载文字："加载中..."
- 长时间加载显示进度条

**Toast提示：**

- 显示在页面顶部中央
- 自动消失（3秒）
- 成功/错误/警告有不同颜色

**确认对话框：**

- 居中显示
- 遮罩层半透明黑色
- 标题、内容、操作按钮清晰分离
- 危险操作用红色按钮

### 响应式设计

**断点：**

- 大屏：≥ 1440px（显示更多内容）
- 桌面：≥ 1024px（标准布局）
- 小屏：< 1024px（简化布局，侧边栏自动隐藏）

**适配策略：**

- 优先保证桌面端体验（≥ 1024px）
- 小屏幕下：
  - 侧边栏改为抽屉式
  - 卡片改为单列显示
  - 减少内边距
  - 搜索框占满宽度

### 动画和过渡

**使用场景：**

- 页面切换：淡入淡出（300ms）
- 卡片悬停：阴影变化（200ms）
- 按钮点击：缩放效果（100ms）
- 侧边栏展开：滑动动画（300ms）
- 内容加载：淡入（400ms）

**原则：**

- 不要过度使用动画
- 保持流畅（60fps）
- 动画时长控制在100-400ms之间
