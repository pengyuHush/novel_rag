## 一、功能模块梳理

### 1.1 模块总览

根据前端需求和 MVP 阶段的实际情况，后端系统划分为 **5 个核心功能模块**：

```
后端系统（简化版）
├── 1. 小说管理模块 (P0)
├── 2. 文本处理模块 (P0)
├── 3. RAG 搜索与问答模块 (P0 核心)
├── 4. 人物关系图谱模块 (P1)
└── 5. 系统管理模块 (P2)
```

### 1.2 详细功能分解

#### 模块 1：小说管理模块

**核心功能**：
- ✅ 小说元数据 CRUD（创建、读取、更新、删除）
- ✅ 小说列表查询（分页、排序、筛选）
- ✅ 小说搜索（书名、作者）
- ✅ 标签管理（简化为小说表字段）
- ✅ 文件上传和管理

**技术要点**：
- 简化的数据库表设计（SQLite）
- JSON 字段存储标签

**涉及 API**：
- `GET /api/v1/novels`（列表查询）

- `GET /api/v1/novels/{novelId}`（获取详情）

#### 模块 5：RAG 搜索与问答模块 ⭐⭐⭐ 核心中的核心

**核心功能**：
- ✅ 自然语言问题理解
- ✅ 问题向量化（Embedding）
- ✅ 向量相似度检索（Top-K）
- ✅ 上下文构建（Prompt Engineering）
- ✅ LLM 答案生成（智谱 GLM-4）
- ✅ 答案后处理和引用标注
- ✅ 搜索历史记录
- ✅ 多小说联合检索

**技术要点**：
- LangChain：文本处理和检索
- LangGraph：Agent 工作流编排
- 智谱 GLM-4：答案生成
- Qdrant：向量检索
- Prompt 模板设计

**RAG 流程**：
```
用户问题
  ↓
1. 问题预处理（去除无关词、意图识别）
  ↓
2. 问题向量化（智谱 Embedding API）
  ↓
3. 向量检索（Qdrant similarity_search, top_k=5）
  ↓
4. 重排序（可选：使用 BGE Reranker）
  ↓
5. 上下文构建
   Prompt = f"""
   基于以下小说内容回答问题：
   
   相关内容：
   {context_chunks}
   
   问题：{user_question}
   
   要求：
   1. 基于提供的内容回答
   2. 如果内容不足以回答，明确说明
   3. 引用具体章节和段落
   """
  ↓
6. 调用 LLM（智谱 GLM-4）
  ↓
7. 答案解析和引用提取
  ↓
8. 返回结果（answer + references）
```

**LangChain 集成**：
```python
from langchain_community.vectorstores import Qdrant
from langchain_community.embeddings import ZhipuAIEmbeddings
from langchain_community.chat_models import ChatZhipuAI
from langchain.chains import RetrievalQA
from langgraph.prebuilt import create_react_agent

# 1. 初始化 Embedding（使用最新的 embedding-3）
embeddings = ZhipuAIEmbeddings(
    api_key=ZHIPU_API_KEY,
    model="embedding-3"
)

# 2. 初始化向量数据库
vectorstore = Qdrant(
    client=qdrant_client,
    collection_name="novel_embeddings",
    embeddings=embeddings
)

# 3. 初始化 LLM（使用 GLM-4.6，模型名为 glm-4-plus）
llm = ChatZhipuAI(
    api_key=ZHIPU_API_KEY,
    model="glm-4-plus",  # GLM-4.6
    temperature=0.7
)

# 4. 创建检索链
retriever = vectorstore.as_retriever(search_kwargs={"k": 5})
qa_chain = RetrievalQA.from_chain_type(
    llm=llm,
    chain_type="stuff",
    retriever=retriever,
    return_source_documents=True
)

# 5. 使用 LangGraph 创建 Agent（可选，用于复杂任务）
from langchain.tools import Tool

tools = [
    Tool(
        name="NovelSearch",
        func=qa_chain.run,
        description="搜索小说内容并回答问题"
    )
]

agent = create_react_agent(llm, tools)
```

**涉及 API**：
- `POST /search`（搜索问答）
- `GET /search/history`（搜索历史）
- `DELETE /search/history`（清空历史）
- `DELETE /search/history/{historyId}`（删除单条）

**优先级**：⭐⭐⭐⭐⭐ P0（核心）

**实现方案**：

**方案一：基于 Prompt 的实体和关系抽取（推荐，简单高效）**

```python
# 1. 人物提取 Prompt
character_extraction_prompt = """
请从以下小说片段中提取所有人物姓名：

{text_chunk}

要求：
1. 只提取人名，不要描述
2. 去除重复
3. 每个人名一行
4. 如有别名，用"|"分隔，如：萧炎|炎帝

输出格式：
萧炎|炎帝
药尘|药老
"""

# 2. 关系提取 Prompt
relationship_extraction_prompt = """
请分析以下小说片段中的人物关系：

{text_chunk}

已知人物列表：
{character_list}

要求：
1. 提取人物之间的关系
2. 关系类型：family, friend, enemy, master-disciple, lover, ally, rival
3. 给出关系强度（1-10）
4. 提供支持证据（原文片段）

输出 JSON 格式：
[
  {
    "source": "药尘",
    "target": "萧炎",
    "type": "master-disciple",
    "strength": 9,
    "evidence": "原文片段..."
  }
]
"""

# 3. 使用 LangChain + GLM-4 处理
from langchain.prompts import PromptTemplate
from langchain.chains import LLMChain

llm = ChatZhipuAI(api_key=ZHIPU_API_KEY, model="glm-4")

# 人物提取
character_chain = LLMChain(
    llm=llm,
    prompt=PromptTemplate.from_template(character_extraction_prompt)
)

# 关系提取
relation_chain = LLMChain(
    llm=llm,
    prompt=PromptTemplate.from_template(relationship_extraction_prompt)
)
```

**处理流程**：
```
小说文本
  ↓
1. 文本分段（按章节或固定长度）
  ↓
2. 对每段调用 GLM-4 提取人物
  ↓
3. 人物去重和合并（相似度匹配）
  ↓
4. 统计人物出场次数
  ↓
5. 对每段调用 GLM-4 提取关系
  ↓
6. 关系聚合和强度计算
  ↓
7. 构建图谱数据结构
  ↓
8. 存储到数据库
  ↓
完成
```

**涉及 API**：
- `GET /api/v1/novels/{novelId}/graph`（获取图谱）
- `POST /api/v1/novels/{novelId}/graph`（生成图谱）
- `DELETE /api/v1/novels/{novelId}/graph`（删除图谱）
- `GET /api/v1/novels/{novelId}/characters`（人物列表）
- `GET /api/v1/novels/{novelId}/characters/{characterId}`（人物详情）

### 2.2 关键技术决策

#### 为什么选择 Qdrant 而非 Milvus？

| 对比项 | Qdrant | Milvus |
|--------|--------|--------|
| 部署复杂度 | ⭐⭐⭐⭐⭐ 简单（单进程） | ⭐⭐⭐ 复杂（多组件） |
| Python 集成 | ⭐⭐⭐⭐⭐ 原生支持 | ⭐⭐⭐⭐ 需要额外配置 |
| 性能 | ⭐⭐⭐⭐ 优秀（百万级） | ⭐⭐⭐⭐⭐ 卓越（千万级+） |
| 资源占用 | ⭐⭐⭐⭐⭐ 轻量 | ⭐⭐⭐ 较重 |
| 适用场景 | 中小规模（本项目） | 大规模生产 |

**结论**：本项目为中小规模应用，Qdrant 更适合快速开发和部署。

#### 为什么使用 Redis 而非 RabbitMQ？

- Redis 可同时作为缓存和消息代理，简化架构
- 本项目任务队列需求不复杂，Redis 足够
- 减少运维复杂度

#### 智谱 GLM-4.6 优势

- ✅ 最新版本，性能和效果更优
- ✅ 中文语料训练，中文理解能力强
- ✅ API 调用简单，无需本地部署
- ✅ 价格合理（Embedding-3: ¥0.0005/千tokens，GLM-4.6: ¥0.05/千tokens）
- ✅ 配套 Embedding-3 模型，向量语义一致性好
- ✅ 支持更长的上下文窗口（128K tokens）

```yaml
    networks:
      - novel_rag_network

      - qdrant_data:/qdrant/storage

  #     # mysql (已移除，使用SQLite)
```

### 5.3 智谱 API Key 获取和配置

#### 获取步骤

1. **注册账号**
   - 访问 [智谱AI开放平台](https://open.bigmodel.cn/)
   - 使用手机号注册账号

2. **实名认证**
   - 完成个人实名认证（需要上传身份证）
   - 认证通过后可获得免费额度

3. **创建 API Key**
   - 登录后进入控制台
   - 点击"API Keys"菜单
   - 点击"创建新的 API Key"
   - 复制生成的 Key（只显示一次，注意保存）

4. **充值（可选）**
   - 新用户有免费额度（约 100 万 tokens）
   - 用完后需要充值（支持微信、支付宝）
   - 最低充值 10 元

#### 配置到项目

在项目根目录创建 `.env` 文件：

```bash
# .env 文件

# 智谱 API Key（必填）
ZHIPU_API_KEY=your_zhipu_api_key_here

# 数据库配置（SQLite）
DATABASE_URL=sqlite+aiosqlite:///./novel_rag.db

# Redis 配置
REDIS_URL=redis://localhost:6379/0

# Qdrant 配置
QDRANT_HOST=localhost
QDRANT_PORT=6333

# 应用配置
APP_NAME=小说RAG分析系统
APP_VERSION=1.0.0
DEBUG=True
LOG_LEVEL=INFO

# 文件上传配置
MAX_FILE_SIZE=52428800  # 50MB
UPLOAD_DIR=./uploads

# RAG 配置
EMBEDDING_MODEL=embedding-3
CHAT_MODEL=glm-4-plus
CHUNK_SIZE=500
CHUNK_OVERLAP=50
TOP_K=5

# Celery 配置
CELERY_BROKER_URL=redis://:redis_password_2024@localhost:6379/1
CELERY_RESULT_BACKEND=redis://:redis_password_2024@localhost:6379/2
```

**安全提示**：
- ⚠️ **不要将 `.env` 文件提交到 Git**
- 在 `.gitignore` 中添加 `.env`
- 生产环境使用环境变量或密钥管理服务

```python
# app/services/rag_service.py

from typing import List, Dict, Any
from langchain_community.vectorstores import Qdrant
from langchain_community.embeddings import ZhipuAIEmbeddings
from langchain_community.chat_models import ChatZhipuAI
from langchain.prompts import PromptTemplate
from langchain.chains import RetrievalQA
from qdrant_client import QdrantClient
from app.core.config import settings
from app.schemas.search import SearchRequest, SearchResponse, SearchReference


class RAGService:
    """RAG 搜索服务"""
    
    def __init__(self):
        # 初始化 Qdrant 客户端
        self.qdrant_client = QdrantClient(
            host=settings.QDRANT_HOST,
            port=settings.QDRANT_PORT
        )
        
        # 初始化 Embedding 模型（使用 embedding-3）
        self.embeddings = ZhipuAIEmbeddings(
            api_key=settings.ZHIPU_API_KEY,
            model="embedding-3"  # 智谱最新 Embedding 模型
        )
        
        # 初始化向量数据库
        self.vectorstore = Qdrant(
            client=self.qdrant_client,
            collection_name="novel_embeddings",
            embeddings=self.embeddings
        )
        
        # 初始化 LLM（使用 GLM-4.6 即 glm-4-plus）
        self.llm = ChatZhipuAI(
            api_key=settings.ZHIPU_API_KEY,
            model="glm-4-plus",  # GLM-4.6 对应的模型名
            temperature=0.7
        )
        
        # 定义 Prompt 模板
        self.prompt_template = PromptTemplate(
            input_variables=["context", "question"],
            template="""你是一个专业的小说分析助手。请基于以下小说内容回答用户的问题。

相关内容：
{context}

问题：{question}

要求：
1. 基于提供的内容进行回答
2. 如果内容不足以回答问题，请明确说明
3. 尽可能引用具体的章节和段落
4. 回答要准确、详细、有条理

回答："""
        )
    
    async def search(self, request: SearchRequest) -> SearchResponse:
        """执行 RAG 搜索"""
        import time
        start_time = time.time()
        
        # 1. 构建检索器（支持按小说 ID 筛选）
        search_kwargs = {"k": request.top_k}
        
        if request.novel_ids:
            # 添加过滤条件
            search_kwargs["filter"] = {
                "must": [
                    {
                        "key": "novel_id",
                        "match": {"any": request.novel_ids}
                    }
                ]
            }
        
        retriever = self.vectorstore.as_retriever(search_kwargs=search_kwargs)
        
        # 2. 创建 QA 链
        qa_chain = RetrievalQA.from_chain_type(
            llm=self.llm,
            chain_type="stuff",
            retriever=retriever,
            return_source_documents=True,
            chain_type_kwargs={"prompt": self.prompt_template}
        )
        
        # 3. 执行查询
        result = await qa_chain.ainvoke({"query": request.query})
        
        # 4. 提取答案和来源文档
        answer = result["result"]
        source_docs = result["source_documents"]
        
        # 5. 构建引用信息
        references = []
        if request.include_references:
            for doc in source_docs:
                metadata = doc.metadata
                references.append(SearchReference(
                    novel_id=metadata.get("novel_id"),
                    novel_title=metadata.get("novel_title"),
                    chapter_id=metadata.get("chapter_id"),
                    chapter_title=metadata.get("chapter_title"),
                    chapter_number=metadata.get("chapter_number"),
                    paragraph_index=metadata.get("paragraph_index"),
                    content=doc.page_content[:200],  # 截取前200字
                    relevance_score=metadata.get("score", 0.0),
                    start_position=metadata.get("start_position"),
                    highlight_text=self._extract_highlight(doc.page_content, request.query)
                ))
        
        # 6. 计算置信度（基于检索分数）
        confidence = self._calculate_confidence(source_docs)
        
        # 7. 生成相关问题推荐
        related_questions = await self._generate_related_questions(request.query, answer)
        
        # 8. 构建响应
        search_time = (time.time() - start_time) * 1000  # 转换为毫秒
        
        response = SearchResponse(
            query=request.query,
            answer=answer,
            confidence=confidence,
            references=references,
            related_questions=related_questions,
            search_time=search_time
        )
        
        # 9. 保存搜索历史（如果需要）
        if request.save_history:
            await self._save_search_history(request, response)
        
        return response
    
    def _extract_highlight(self, text: str, query: str) -> str:
```

### 6.2 文本向量化核心代码示例

```python
# app/services/text_processing_service.py

import chardet
import re
from typing import List, Tuple
from langchain.text_splitter import RecursiveCharacterTextSplitter
from qdrant_client.models import Distance, VectorParams, PointStruct
from app.core.config import settings
from app.services.rag_service import RAGService


class TextProcessingService:
    """文本处理服务"""
    
    def __init__(self):
        self.rag_service = RAGService()
        self.text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=settings.CHUNK_SIZE,
            chunk_overlap=settings.CHUNK_OVERLAP,
            separators=["\n\n", "\n", "。", "！", "？", "；", " ", ""],
            length_function=len
        )
    
    def detect_encoding(self, file_content: bytes) -> str:
        """检测文件编码"""
        result = chardet.detect(file_content)
        encoding = result['encoding']
        confidence = result['confidence']
        
        # 如果置信度低于 0.7，尝试常见编码
        if confidence < 0.7:
            for enc in ['utf-8', 'gbk', 'gb2312']:
                try:
                    file_content.decode(enc)
                    return enc
                except UnicodeDecodeError:
                    continue
        
        return encoding
    
    def validate_content(self, text: str) -> Tuple[bool, str]:
        """验证文本内容"""
        import jieba
        
        # 1. 检查最少字数
        word_count = len(text)
        if word_count < 1000:
            return False, f"文本过短，仅 {word_count} 字，至少需要 1000 字"
        
        # 2. 检查中文占比
        chinese_chars = len(re.findall(r'[\u4e00-\u9fff]', text))
        chinese_ratio = chinese_chars / word_count if word_count > 0 else 0
        
        if chinese_ratio < 0.6:
            return False, f"中文字符占比过低（{chinese_ratio:.1%}），至少需要 60%"
        
        return True, "验证通过"
    
    def detect_chapters(self, text: str, pattern: str = None) -> List[Dict]:
        """自动识别章节"""
        if pattern is None:
            # 默认章节识别正则
            pattern = r'第[零一二三四五六七八九十百千万\d]+[章回节话].*'
        
        chapters = []
        lines = text.split('\n')
        
        for i, line in enumerate(lines):
            line = line.strip()
            if re.match(pattern, line):
                # 找到章节标题
                chapters.append({
                    'title': line,
                    'line_number': i,
                    'position': text.index(line)
                })
        
        # 计算每章的起止位置
        for i, chapter in enumerate(chapters):
            chapter['start_position'] = chapter['position']
            if i < len(chapters) - 1:
                chapter['end_position'] = chapters[i + 1]['position']
            else:
                chapter['end_position'] = len(text)
            
            # 计算字数
            chapter_text = text[chapter['start_position']:chapter['end_position']]
            chapter['word_count'] = len(chapter_text)
        
        return chapters
    
    async def vectorize_and_store(
        self, 
        novel_id: str, 
        text: str, 
        chapters: List[Dict]
    ) -> int:
        """文本向量化并存储到 Qdrant"""
        
        # 1. 确保 Collection 存在
        await self._ensure_collection()
        
        # 2. 分段
        chunks = self.text_splitter.split_text(text)
        
        # 3. 为每个 chunk 添加元数据
        points = []
        for i, chunk in enumerate(chunks):
            # 确定 chunk 所属章节
            chunk_position = text.index(chunk)
            chapter_info = self._find_chapter(chunk_position, chapters)
            
            # 生成 embedding
            embedding = await self.rag_service.embeddings.aembed_query(chunk)
            
            # 构建 point
            point = PointStruct(
                id=i,
                vector=embedding,
                payload={
                    "novel_id": novel_id,
                    "chunk_index": i,
                    "content": chunk,
                    "chapter_id": chapter_info.get("id"),
                    "chapter_title": chapter_info.get("title"),
                    "chapter_number": chapter_info.get("number"),
                    "paragraph_index": i,
                    "start_position": chunk_position
                }
            )
            points.append(point)
        
        # 4. 批量插入 Qdrant
        self.rag_service.qdrant_client.upsert(
            collection_name="novel_embeddings",
            points=points
        )
        
        return len(chunks)
    
    async def _ensure_collection(self):
        """确保 Qdrant Collection 存在"""
        collections = self.rag_service.qdrant_client.get_collections().collections
        collection_names = [c.name for c in collections]
        
        if "novel_embeddings" not in collection_names:
            self.rag_service.qdrant_client.create_collection(
                collection_name="novel_embeddings",
                vectors_config=VectorParams(
                    size=2048,  # 智谱 embedding-3 的向量维度
                    distance=Distance.COSINE
                )
            )
    
    def _find_chapter(self, position: int, chapters: List[Dict]) -> Dict:
        """根据位置找到所属章节"""
        for chapter in chapters:
            if chapter['start_position'] <= position < chapter['end_position']:
                return chapter
        return {}
```
