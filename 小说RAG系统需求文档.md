# 小说RAG系统功能需求文档（MVP版）

## 1. 项目概述

### 1.1 系统简介

小说RAG系统是一个专注于**中文长篇小说**的智能检索增强生成平台，旨在帮助用户快速理解小说内容、精准定位关键信息、分析人物关系。系统采用**网页端**部署方式，支持单部或多部关联小说的联合分析。

### 1.2 目标用户

- 小说爱好者：深度阅读和理解小说内容
- 内容创作者：参考学习和灵感获取
- 文学爱好者：进行文本分析和探讨

### 1.3 核心特点

- 专注中文小说，支持TXT文本格式
- 支持数百万字级别的超长篇小说（300万字以上）
- 智能化的内容检索和问答能力
- 精准的原文定位和引用功能
- 自动生成人物关系网络图谱
- 纯网页端访问，无需安装

## 2. 核心功能需求

### 2.1 文本导入与管理

#### 2.1.1 文本导入

**功能描述：** 支持中文TXT格式小说文本的导入

**详细需求：**

- 支持文件格式：TXT（纯文本）
- 支持编码格式：UTF-8、GBK、GB2312
- 支持单文件导入和批量导入（多部小说）
- 支持拖拽上传和文件选择两种方式
- 自动检测文件编码并正确解析
- 支持超大文件导入（300万字以上）
- 显示导入进度条和状态提示（文件大小、字数统计）
- 导入完成后显示文本基本信息预览

#### 2.1.2 小说信息管理

**功能描述：** 管理已导入小说的基本信息

**详细需求：**

- 录入或编辑小说基本信息：
  - 书名（必填）
  - 作者（可选）
  - 简介（可选）
  - 自定义标签（可选）
- 建立小说之间的关联关系：
  - 标记系列小说（如：系列第1部、第2部）
  - 标记相关小说（同一世界观、角色关联等）
- 显示小说列表，包含书名、作者、字数、导入时间
- 支持小说删除功能（需二次确认）
- 显示存储空间使用情况

#### 2.1.3 章节识别与管理

**功能描述：** 自动识别中文小说的章节结构

**详细需求：**

- 自动识别常见的中文章节标题格式：
  - "第X章"、"第X回"
  - "第X节"、"第X话"
  - "Chapter X"、"章节 X"
  - 支持章节标题后带名称（如："第一章 开端"）
- 生成章节目录树，显示：
  - 章节序号
  - 章节标题
  - 章节字数
  - 起止位置
- 支持预览章节内容
- 如识别结果不准确，支持手动调整章节边界
- 处理无章节划分的小说（按段落或字数自动分段）

---

### 2.2 智能检索与问答

#### 2.2.1 搜索功能

**功能描述：** 提供基于关键词和语义的搜索能力

**详细需求：**

- **关键词搜索：**
  - 支持输入单个或多个关键词
  - 支持精确匹配和模糊匹配
  - 支持设置搜索范围（全部小说 / 指定小说 / 指定章节范围）
  
- **语义搜索：**
  - 支持自然语言问题输入（如："XX角色在哪里出现过？"）
  - 理解用户问题的意图，检索相关内容
  - 支持中文分词和语义理解

#### 2.2.2 智能问答

**功能描述：** 基于小说内容回答用户问题

**详细需求：**

- **支持的问题类型：**
  - 事实性问题："XX角色是谁？" "XX的身份是什么？"
  - 情节问题："XX事件发生在哪里？" "XX和YY什么时候相遇的？"
  - 关系问题："XX和YY是什么关系？"
  - 总结问题："总结第X章的内容"
  - 对比问题："比较XX和YY的特点"

- **回答生成要求：**
  - 生成准确、简洁的自然语言回答
  - 回答必须基于原文内容，不得编造信息
  - 如果无法从文本中找到确切答案，需明确说明
  - 回答长度适中（100-300字为宜）

#### 2.2.3 搜索结果展示

**功能描述：** 清晰展示搜索和问答结果，提供原文引用

**详细需求：**

- **结果布局：**
  - 顶部显示生成的回答内容（醒目展示）
  - 下方显示参考段落列表（按相关度排序）

- **参考段落信息（每条包含）：**
  - 来源小说名称
  - 章节信息（格式："第X章 章节标题"）
  - 段落位置（第X段，或字符位置范围）
  - 原文摘录（200-500字，关键词高亮显示）
  - 相关度指示（可用百分比或星级）

- **交互功能：**
  - 点击参考段落可展开显示更多上下文（前后各1-2段）
  - 点击"跳转"按钮可查看完整章节内容
  - 支持复制回答和原文内容
  - 高亮显示匹配的关键词或相关语句

#### 2.2.4 多小说联合检索

**功能描述：** 支持在多部小说中同时进行检索

**详细需求：**

- 默认在已建立关联关系的小说组内检索
- 可手动选择检索范围（勾选指定的小说）
- 结果按小说分组显示，每组内按相关度排序
- 自动识别跨小说的相同角色名称
- 支持跨小说的关联分析（如："XX角色在两部小说中有何不同表现？"）

#### 2.2.5 搜索历史

**功能描述：** 记录和管理用户的搜索历史

**详细需求：**

- 自动保存每次搜索的问题和生成的答案
- 显示搜索历史列表（时间倒序）
- 支持点击历史记录查看完整问答结果
- 支持删除单条历史或清空全部历史
- 历史记录保存在浏览器本地存储

---

### 2.3 人物关系图谱

#### 2.3.1 关系图谱生成

**功能描述：** 自动识别并生成小说中的人物关系网络

**详细需求：**

- **自动识别：**
  - 自动提取小说中的人物名称（支持中文姓名识别）
  - 分析人物之间的共现关系和互动频率
  - 识别常见关系类型（家人、朋友、敌对、师徒等）
  - 根据出现频率判断主要人物和次要人物

- **图谱构建：**
  - 人物作为节点，关系作为边
  - 节点大小反映人物重要性（出现频率）
  - 不同关系类型使用不同颜色或线型区分
  - 自动布局，保持图形美观和可读性

#### 2.3.2 图谱展示

**功能描述：** 以可交互的方式展示人物关系网络

**详细需求：**

- 使用力导向图或网络图展示
- 节点显示人物名称
- 鼠标悬停显示人物简要信息
- 不同关系类型用不同颜色标识（配有图例）
- 支持缩放、拖拽、平移操作
- 支持点击节点高亮该人物的所有关系
- 支持筛选功能（如：只显示主要人物）

#### 2.3.3 关系详情

**功能描述：** 查看人物关系的详细信息

**详细需求：**

- 点击节点（人物）显示：
  - 人物名称
  - 出现次数统计
  - 主要出场章节列表
  - 关系列表（与其他人物的关系）
  
- 点击边（关系线）显示：
  - 关系类型
  - 共同出现的章节
  - 代表性原文片段（展示关系的典型场景）
  - 支持点击跳转到原文位置

#### 2.3.4 图谱导出

**功能描述：** 导出人物关系图谱

**详细需求：**

- 支持导出为图片格式（PNG、JPG）
- 可选图片尺寸和分辨率
- 保留图例和标题信息

## 3. 用户操作流程

### 3.1 核心使用场景：深度阅读辅助

**场景描述：** 用户在阅读长篇小说过程中，利用系统辅助理解内容

**操作流程：**

1. **导入小说**
   - 用户上传小说TXT文件
   - 系统自动识别编码和章节结构
   - 用户确认或调整章节识别结果
   - 填写小说基本信息（书名、作者等）

2. **浏览章节**
   - 查看自动生成的章节目录
   - 预览章节内容

3. **提出问题**
   - 在阅读过程中遇到疑问时，在搜索框输入问题
   - 例如：
     - "XX角色第一次出现在哪里？"
     - "XX和YY的关系是什么？"
     - "总结一下第10章发生了什么"

4. **查看回答和原文**
   - 系统生成回答并展示在顶部
   - 下方列出相关的原文段落
   - 点击查看更多上下文或跳转到完整章节
   - 复制重要内容供后续参考

5. **查看人物关系**
   - 点击进入人物关系图谱页面
   - 查看主要人物的关系网络
   - 点击人物或关系线查看详细信息
   - 通过图谱理解复杂的人物关系

6. **回顾历史**
   - 查看搜索历史，回顾之前的问答
   - 根据需要重新查看或删除历史记录

### 3.2 扩展场景：多部小说对比分析

**场景描述：** 用户同时分析一个系列的多部小说

**操作流程：**

1. 批量导入系列小说的多个TXT文件
2. 在小说管理中建立它们的系列关联关系
3. 进行跨小说的搜索和问答
   - 例如："XX角色在第二部中性格有什么变化？"
4. 查看综合的人物关系图谱
5. 导出图谱进行展示或分享

## 4. 非功能需求

### 4.1 性能要求

**超长文本支持：**

- 必须支持300万字以上的超长篇小说（约5-10MB的TXT文件）
- 单次导入300万字小说的处理时间不超过2分钟
- 导入后的索引构建时间不超过5分钟

**响应时间：**

- 常规问答（简单事实性问题）响应时间不超过3秒
- 复杂问答（总结、对比类问题）响应时间不超过8秒
- 关键词搜索响应时间不超过2秒
- 人物关系图谱生成时间不超过15秒（300万字小说）

**并发与稳定性：**

- 支持至少20部小说同时管理
- 支持多部小说的联合检索（最多5部同时检索）
- 长时间使用不出现内存泄漏或性能下降

### 4.2 中文文本处理要求

**编码兼容性：**

- 准确识别和转换UTF-8、GBK、GB2312等常见编码
- 处理编码错误或混合编码的容错能力

**中文特性支持：**

- 准确的中文分词能力
- 中文人名识别准确率不低于85%
- 支持中文常见章节标题格式识别
- 处理中文标点符号和特殊字符

**文本预处理：**

- 自动清理多余空行和空格
- 识别并保留段落结构
- 处理繁简体混排的情况

### 4.3 可用性要求

**易用性：**

- 界面简洁清晰，符合中文用户使用习惯
- 新用户可在5分钟内完成首次导入和搜索
- 关键功能入口醒目，操作逻辑清晰
- 所有操作提供明确的反馈提示

**容错性：**

- 文件导入失败时给出明确的错误提示和解决建议
- 章节识别错误时允许用户手动调整
- 搜索无结果时提供有用的提示或建议
- 系统异常时不丢失用户数据

### 4.4 浏览器兼容性

**支持的浏览器：**

- Google Chrome（推荐，最新版本及近3个版本）
- Microsoft Edge（最新版本及近3个版本）
- Firefox（最新版本及近3个版本）
- Safari（macOS，最新版本及近2个版本）

**最低浏览器要求：**

- 支持HTML5和CSS3
- 支持ES6+ JavaScript
- 支持WebAssembly（用于高性能文本处理）
- 本地存储（localStorage）容量至少50MB

**响应式设计：**

- 主要针对桌面端浏览器优化（屏幕宽度 ≥ 1024px）
- 在较大屏幕（≥ 1440px）提供更好的显示效果
- 不强制要求移动端支持，但基础功能应可用

### 4.5 数据安全与隐私

**数据存储：**

- 用户导入的小说内容存储在浏览器本地（IndexedDB）
- 搜索历史存储在浏览器本地（localStorage）
- 不上传用户小说内容到服务器（除非用户明确选择云存储功能）

**数据管理：**

- 用户可随时删除已导入的小说及相关数据
- 提供"清空所有数据"功能（需二次确认）
- 支持数据导出备份（可选功能）

**隐私保护：**

- 不收集用户的阅读行为数据
- 不分享或出售用户数据
- 明确的隐私政策说明

## 5. 界面需求

### 5.1 页面结构

系统包含以下核心页面：

**1. 首页 / 小说管理页**

- 显示已导入的小说列表
- 提供导入新小说的入口
- 显示每部小说的基本信息（书名、作者、字数、导入时间）
- 提供小说的操作入口（搜索、查看图谱、删除等）
- 显示系统使用统计（已导入小说数、总字数、存储空间）

**2. 搜索与问答页**

- 顶部：搜索框（支持自然语言输入）
- 搜索框下方：搜索范围选择（全部小说 / 指定小说）
- 中间主区域：显示回答和参考段落
- 侧边栏（可选）：搜索历史列表
- 底部：页码或加载更多按钮

**3. 人物关系图谱页**

- 主区域：可交互的关系网络图
- 左侧：人物筛选和图例
- 右侧：选中人物或关系的详情面板
- 顶部工具栏：缩放、导出、回到搜索等操作

**4. 章节浏览页（可选）**

- 左侧：章节目录树
- 右侧：章节内容显示区
- 支持从搜索结果直接跳转到指定章节位置

### 5.2 界面风格

**视觉设计：**

- 简洁现代的设计风格
- 以白色或浅色为主色调（减轻长时间阅读疲劳）
- 突出内容，减少视觉干扰
- 合理的间距和排版，便于阅读

**字体与排版：**

- 使用适合中文阅读的字体（如思源宋体、苹方、微软雅黑等）
- 正文字号14-16px，行高1.6-1.8
- 标题和正文有明显的层级区分

**交互反馈：**

- 按钮和链接有明确的悬停和点击效果
- 加载过程显示进度指示器
- 操作成功或失败有toast提示
- 重要操作（如删除）有确认对话框

## 6. 技术约束

### 6.1 输入约束

- 仅支持TXT格式文本文件
- 文件编码：UTF-8、GBK、GB2312
- 单个文件大小建议不超过20MB（约400万字）
- 文件内容必须为纯文本，不支持富文本或特殊格式

### 6.2 处理约束

- 系统针对中文文本优化，其他语言可能效果不佳
- 人物关系识别基于统计和规则，准确率受文本质量影响
- 问答质量依赖原文信息的完整性和明确性

### 6.3 浏览器约束

- 需要现代浏览器支持（见4.4节）
- 建议使用高性能设备以获得更好体验
- 浏览器本地存储空间限制可能影响可导入的小说数量

## 7. 未来扩展方向

虽然当前版本聚焦核心功能，但以下是可考虑的未来扩展方向：

- **更多格式支持：** EPUB、PDF等格式的小说导入
- **云端存储：** 支持小说内容云端保存，跨设备访问
- **笔记功能：** 用户可在阅读时添加个人笔记和标注
- **导出能力增强：** 支持导出分析报告（PDF、Word等格式）
- **更多可视化：** 情节时间线、角色出场统计等图表
- **移动端优化：** 提供更好的移动端阅读和搜索体验
- **多用户协作：** 支持分享和讨论功能
