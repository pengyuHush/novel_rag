## 🔴 P0 严重问题：URL 前缀不匹配

### 问题描述

**前端 API 基础路径**：
```typescript
// frontend/src/utils/api.ts
const API_BASE_URL = 'http://localhost:8000';

// 实际调用示例
await fetch(`${API_BASE_URL}/novels`)           // ❌ 错误
await fetch(`${API_BASE_URL}/search`)            // ❌ 错误
await fetch(`${API_BASE_URL}/novels/${id}/graph`) // ❌ 错误
```

**后端 API 规范路径**：
```yaml
# backend_api_specification.yaml
servers:
  - url: http://localhost:8000/v1  # ✅ 正确（注意 /v1）

paths:
  /novels:              # 实际完整路径：/v1/novels
  /search:              # 实际完整路径：/v1/search
  /novels/{novelId}/graph:  # 实际完整路径：/v1/novels/{novelId}/graph
```

### 影响范围

**所有接口都受影响**，前端无法与后端正常通信。

### 解决方案

**方案1：修改前端基础路径（推荐）**
```typescript
// frontend/src/utils/api.ts
const API_BASE_URL = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000/api/v1';
//                                                        修改为 ↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑↑
```

**方案2：修改后端路由（不推荐）**
```python
# backend/app/main.py
# 不推荐：破坏了 API 版本管理
app.include_router(api_router, prefix="")  # 移除 /api/v1 前缀
```

**推荐：方案1**，保持 RESTful API 最佳实践（包含版本号）。

## 🟡 P1 重要问题：接口参数不一致

### 问题 1：小说上传接口不一致

#### 前端实现

```typescript
// frontend/src/utils/api.ts:74-92
async uploadNovel(file: File): Promise<{ novelId: string }> {
  const formData = new FormData();
  formData.append('file', file);

  const response = await fetch(`${API_BASE_URL}/novels`, {
    method: 'POST',
    body: formData,  // ❌ 直接上传文件
  });
  
  return response.json();
}
```

**前端期望**：
- 端点：`POST /novels`
- 请求体：`multipart/form-data` 包含文件
- 响应：`{ novelId: string }`

#### 后端 API 规范

**方式1：创建小说记录**（YAML 第 169-226 行）
```yaml
POST /novels
Content-Type: application/json
{
  "title": "斗破苍穹",
  "author": "天蚕土豆",
  "description": "...",
  "tags": ["玄幻", "修仙"]
}
Response:
{
  "message": "小说创建成功",
  "novel": { ...Novel对象 }
}
```

**方式2：上传文件**（YAML 第 330-437 行）
```yaml
POST /novels/{novelId}/upload
Content-Type: multipart/form-data
file: <binary>
Response:
{
  "message": "文件上传成功",
  "novelId": "novel_123",
  "status": "processing"
}
```

#### 不一致点

1. **路径不同**：
   - 前端：`POST /novels`（期望直接上传文件）
   - 后端：`POST /novels`（创建元数据）+ `POST /novels/{id}/upload`（上传文件）

2. **工作流程不同**：
   - 前端：一步完成（文件上传+元数据）
   - 后端：两步流程（先创建记录，再上传文件）

3. **响应格式不同**：
   - 前端期望：`{ novelId: string }`
   - 后端返回：`{ message: string, novel: Novel }`

#### 建议解决方案

**方案A：修改前端适配后端两步流程（推荐）**

```typescript
// frontend/src/utils/api.ts
export const novelAPI = {
  // 1. 先创建小说记录
  async createNovel(metadata: {
    title: string;
    author?: string;
    description?: string;
    tags?: string[];
  }): Promise<Novel> {
    return apiRequest<Novel>('/api/v1/novels', {
      method: 'POST',
      body: JSON.stringify(metadata),
    });
  },

  // 2. 再上传文件
  async uploadNovelFile(novelId: string, file: File): Promise<void> {
    const formData = new FormData();
    formData.append('file', file);

    const response = await fetch(`${API_BASE_URL}/api/v1/novels/${novelId}/upload`, {
      method: 'POST',
      body: formData,
    });

    if (!response.ok) {
      throw new APIError('Upload failed', response.status);
    }
  },

  // 3. 封装便捷方法（前端组件使用）
  async uploadNovel(file: File, metadata: {
    title: string;
    author?: string;
    description?: string;
    tags?: string[];
  }): Promise<string> {
    // 先创建小说记录
    const novel = await this.createNovel(metadata);
    
    // 再上传文件
    await this.uploadNovelFile(novel.id, file);
    
    return novel.id;
  }
};
```

**方案B：修改后端支持单步上传（不推荐）**

```yaml
# 不推荐：破坏了关注点分离原则
POST /novels
Content-Type: multipart/form-data
file: <binary>
title: "斗破苍穹"
author: "天蚕土豆"
```

**推荐：方案A**，保持后端设计的关注点分离原则。

### 问题 2：搜索接口参数格式不一致

#### 前端实现

```typescript
// frontend/src/utils/api.ts:126-150
async search(params: {
  query: string;
  novelIds?: string[];
  mode?: 'keyword' | 'semantic';
  topK?: number;
}): Promise<SearchResult> {
  const queryParams = new URLSearchParams();
  queryParams.append('query', params.query);

  if (params.novelIds && params.novelIds.length > 0) {
    params.novelIds.forEach(id => queryParams.append('novelIds', id));
  }

  if (params.mode) {
    queryParams.append('mode', params.mode);
  }

  if (params.topK) {
    queryParams.append('topK', params.topK.toString());
  }

  return apiRequest<SearchResult>(`/search?${queryParams.toString()}`);
  // ❌ 使用 GET 请求 + Query String
}
```

#### 后端 API 规范

```yaml
# backend_api_specification.yaml:826-925
/search:
  post:  # ✅ 使用 POST 方法
    requestBody:
      required: true
      content:
        application/json:
          schema:
            type: object
            properties:
              query: string
              novelIds: array
              topK: integer
              includeReferences: boolean
              saveHistory: boolean
```

#### 不一致点

1. **HTTP 方法不同**：
   - 前端：`GET /search?query=...&novelIds=...`
   - 后端：`POST /search` + JSON body

2. **参数位置不同**：
   - 前端：Query String 参数
   - 后端：Request Body（JSON）

3. **参数命名不同**：
   - 前端：`mode`（keyword/semantic）
   - 后端：无此参数

4. **缺少参数**：
   - 前端缺少：`includeReferences`、`saveHistory`

#### 建议解决方案

**修改前端使用 POST + JSON Body（推荐）**

```typescript
// frontend/src/utils/api.ts
export const searchAPI = {
  async search(params: {
    query: string;
    novelIds?: string[];
    topK?: number;
    includeReferences?: boolean;
    saveHistory?: boolean;
  }): Promise<SearchResult> {
    return apiRequest<SearchResult>('/api/v1/search', {
      method: 'POST',
      body: JSON.stringify({
        query: params.query,
        novelIds: params.novelIds || [],
        topK: params.topK || 5,
        includeReferences: params.includeReferences !== false,
        saveHistory: params.saveHistory !== false,
      }),
    });
  }
};
```

**说明**：
- 使用 POST 方法符合 RESTful 最佳实践（搜索是有副作用的操作，会保存历史记录）
- JSON Body 比 Query String 更适合复杂参数（如数组）
- 移除前端的 `mode` 参数（后端智能判断）

```typescript
async getChapter(novelId: string, chapterId: string): Promise<ChapterContent> {
```

```yaml
# 不推荐：丢失了段落信息

  "novelId": "novel_abc123",
```

```typescript
export const novelAPI = {
  // 新增：创建小说记录
  async createNovel(metadata: Omit<Novel, 'id' | 'content' | 'chapters' | 'wordCount' | 'importedAt'>): Promise<Novel> {
    return apiRequest<{ novel: Novel }>(`/novels`, {
      method: 'POST',
      body: JSON.stringify(metadata),
    }).then(res => res.novel);
  },

  // 修改：上传文件到已创建的小说
  async uploadNovelFile(novelId: string, file: File): Promise<void> {
    const formData = new FormData();
    formData.append('file', file);

    await fetch(`${API_BASE_URL}/novels/${novelId}/upload`, {
      method: 'POST',
      body: formData,
    });
  },

  // 封装：一步完成上传（供组件使用）
  async uploadNovel(file: File, metadata: Omit<Novel, 'id' | 'content' | 'chapters' | 'wordCount' | 'importedAt'>): Promise<string> {
    const novel = await this.createNovel(metadata);
    await this.uploadNovelFile(novel.id, file);
    return novel.id;
  },
  
  // ... 其他方法保持不变
};
```
