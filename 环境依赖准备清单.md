### 2.2 各服务详细配置

#### 服务 1：SQLite 数据库 ✅

- [ ] **SQLite 3.40+** 已安装（Python 内置）
  ```bash
  # SQLite 是 Python 内置的，无需额外安装
  python -c "import sqlite3; print('SQLite version:', sqlite3.sqlite_version)"
  ```

- [ ] **数据库文件准备**
  ```bash
  # 确保项目目录有写权限
  touch backend/novel_rag.db
  ls -la backend/novel_rag.db
  ```

- **配置信息**：
  - 数据库文件：`./backend/novel_rag.db`
  - 连接字符串：`sqlite+aiosqlite:///./novel_rag.db`
  - 无需用户名密码（本地文件数据库）

#### 服务 2：Redis 缓存和消息队列 ✅

- [ ] **Redis 7+** 已启动
  ```bash
  # Docker 方式（推荐）
  docker run -d \
    --name novel_rag_redis \
    -p 6379:6379 \
    -v redis_data:/data \
    redis:7-alpine \
    redis-server --requirepass redis_password_2024 --appendonly yes
  
  # 验证连接
  docker exec -it novel_rag_redis redis-cli -a redis_password_2024 ping
  # 应返回：PONG
  ```

- **配置信息**：
  - 密码：`redis_password_2024`
  - 端口：`6379`
  - 连接字符串：`redis://:redis_password_2024@localhost:6379/0`

#### 服务 3：Qdrant 向量数据库 ✅

- [ ] **Qdrant 1.7+** 已启动
  ```bash
  # Docker 方式（推荐）
  docker run -d \
    --name novel_rag_qdrant \
    -p 6333:6333 \
    -p 6334:6334 \
    -v qdrant_data:/qdrant/storage \
    qdrant/qdrant:v1.7.4
  
  # 验证连接
  curl http://localhost:6333/healthz
  # 应返回：OK
  ```

- **配置信息**：
  - HTTP API 端口：`6333`
  - gRPC API 端口：`6334`
  - 管理界面：http://localhost:6333/dashboard

## ✅ 第五步：环境验证

### 5.1 完整验证脚本

创建 `verify_env.py` 文件：

```python
#!/usr/bin/env python3
"""环境验证脚本"""

import os
import sys
from dotenv import load_dotenv

# 加载环境变量
load_dotenv()

def check_env():
    """检查环境变量"""
    print("📋 检查环境变量...")
    
    required_vars = [
        "ZHIPU_API_KEY",
        "DATABASE_URL",
        "REDIS_URL",
        "QDRANT_HOST",
        "QDRANT_PORT"
    ]
    
    missing = []
    for var in required_vars:
        value = os.getenv(var)
        if not value:
            missing.append(var)
            print(f"  ❌ {var} 未设置")
        else:
            # 隐藏敏感信息
            if "KEY" in var or "PASSWORD" in var:
                display_value = value[:8] + "..." + value[-4:]
            else:
                display_value = value
            print(f"  ✅ {var} = {display_value}")
    
    if missing:
        print(f"\n⚠️  缺少环境变量：{', '.join(missing)}")
        return False
    
    print("\n✅ 环境变量检查通过\n")
    return True


def check_mysql():
    """检查 MySQL 连接"""
    print("🐬 检查 MySQL 连接...")
    try:
        import aiomysql
        import asyncio
        
        async def test_conn():
            conn = await aiomysql.connect(
                host='localhost',
                port=3306,
                user='novel_user',
                password='novel_password_2024',
                db='novel_rag'
            )
            async with conn.cursor() as cursor:
                await cursor.execute("SELECT VERSION()")
                version = await cursor.fetchone()
            conn.close()
            return version[0]
        
        version = asyncio.run(test_conn())
        print(f"  ✅ MySQL 连接成功")
        print(f"  版本：{version}\n")
        return True
    except Exception as e:
        print(f"  ❌ MySQL 连接失败：{e}\n")
        return False


def check_redis():
    """检查 Redis 连接"""
    print("🔴 检查 Redis 连接...")
    try:
        import redis
        
        # 解析 Redis URL
        redis_url = os.getenv("REDIS_URL")
        r = redis.from_url(redis_url)
        
        # 测试连接
        pong = r.ping()
        info = r.info("server")
        
        print(f"  ✅ Redis 连接成功")
        print(f"  版本：{info['redis_version']}\n")
        return True
    except Exception as e:
        print(f"  ❌ Redis 连接失败：{e}\n")
        return False


def check_qdrant():
    """检查 Qdrant 连接"""
    print("🔍 检查 Qdrant 连接...")
    try:
        from qdrant_client import QdrantClient
        
        host = os.getenv("QDRANT_HOST", "localhost")
        port = int(os.getenv("QDRANT_PORT", 6333))
        
        client = QdrantClient(host=host, port=port)
        collections = client.get_collections()
        
        print(f"  ✅ Qdrant 连接成功")
        print(f"  当前集合数：{len(collections.collections)}\n")
        return True
    except Exception as e:
        print(f"  ❌ Qdrant 连接失败：{e}\n")
        return False


def check_zhipu_api():
    """检查智谱 API"""
    print("🤖 检查智谱 GLM API...")
    try:
        from zhipuai import ZhipuAI
        
        api_key = os.getenv("ZHIPU_API_KEY")
        client = ZhipuAI(api_key=api_key)
        
        # 测试 Embedding API（使用最新的 embedding-3）
        response = client.embeddings.create(
            model="embedding-3",
            input="测试文本"
        )
        
        print(f"  ✅ 智谱 API 连接成功")
        print(f"  Embedding 模型：embedding-3")
        print(f"  向量维度：{len(response.data[0].embedding)}\n")
        return True
    except Exception as e:
        print(f"  ❌ 智谱 API 连接失败：{e}\n")
        print(f"  请检查 API Key 是否正确，是否已充值\n")
        return False


def main():
    """主函数"""
    print("=" * 60)
    print("小说 RAG 分析系统 - 环境验证")
    print("=" * 60)
    print()
    
    checks = [
        check_env,
        check_mysql,
        check_redis,
        check_qdrant,
        check_zhipu_api
    ]
    
    results = []
    for check in checks:
        try:
            result = check()
            results.append(result)
        except Exception as e:
            print(f"  ❌ 检查失败：{e}\n")
            results.append(False)
    
    print("=" * 60)
    if all(results):
        print("🎉 所有检查通过，环境配置完成！")
        print("=" * 60)
        return 0
    else:
        print("⚠️  部分检查未通过，请修复后重试")
        print("=" * 60)
        return 1


if __name__ == "__main__":
    sys.exit(main())
```

## 🆘 常见问题

### Q1：MySQL 连接失败

**症状**：`Can't connect to MySQL server`

**解决方案**：
```bash
# 检查容器是否运行
docker ps | grep mysql

# 查看日志
docker logs novel_rag_mysql

# 测试连接
docker exec -it novel_rag_mysql mysql -u novel_user -pnovel_password_2024 -e "SELECT 1;"

# 重启容器
docker restart novel_rag_mysql
```

### Q2：Redis 连接失败

**症状**：`ConnectionRefusedError`

**解决方案**：
```bash
# 检查容器状态
docker ps | grep redis

# 测试连接
docker exec -it novel_rag_redis redis-cli -a redis_password_2024 ping

# 重启
docker restart novel_rag_redis
```

### Q3：Qdrant 连接失败

**症状**：`Cannot connect to host localhost:6333`

**解决方案**：
```bash
# 检查容器
docker ps | grep qdrant

# 查看日志
docker logs novel_rag_qdrant

# 测试 HTTP API
curl http://localhost:6333/healthz

# 重启
docker restart novel_rag_qdrant
```

### Q4：智谱 API 调用失败

**症状**：`Invalid API Key` 或 `Insufficient balance`

**解决方案**：
1. 检查 API Key 是否正确（复制时没有多余空格）
2. 检查账户余额是否充足
3. 检查 API Key 是否已激活
4. 访问控制台确认 Key 状态

### Q5：Poetry 安装依赖慢

**解决方案**：
```bash
# 配置清华镜像源
poetry source add tsinghua https://pypi.tuna.tsinghua.edu.cn/simple/ --priority=primary

# 重新安装
poetry install
```
